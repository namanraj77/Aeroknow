Multi-Vehicle SITL — How PX4 Runs 5 Drones

PX4 SITL supports multi-vehicle operation by running multiple instances of the PX4 binary simultaneously, each on different UDP ports and each spawning a separate Gazebo model instance. Understanding the port scheme is essential before writing any launch code.

Each PX4 instance is launched with an instance number that offsets all its ports. The formula: GCS_port = 14550 + (instance × 10), MAVROS_listen_port = 14540 + instance.

ROS2 Namespacing

Without namespacing, five MAVROS2 nodes would all publish to the same /mavros/state topic — they would overwrite each other. Namespacing prefixes every topic with the drone ID:

Namespacing in ROS2 — The One-Line Version Every ROS2 node launched with namespace='/drone_1' automatically prefixes all its topics, services, and actions with /drone_1/. Your code doesn't change — MAVROS2's /mavros/state becomes /drone_1/mavros/state automatically. The flocking node for drone 1 subscribes to /drone_1/mavros/local_position/pose and publishes to /drone_1/mavros/setpoint_velocity/cmd_vel_unstamped. Five identical flocking nodes, five namespaces, five independent control loops.

STEP 02 Write swarm_launcher.py
Why Python subprocess instead of a ROS2 launch file

Launching 5 PX4 binaries requires setting different environment variables (PX4_SIM_MODEL, PX4_INSTANCE) per process and managing their working directories individually. ROS2 launch files can do this, but the conditional logic (same command, different env vars, different ports) is cleaner in Python with a loop. The launcher uses subprocess.Popen to start each PX4 process non-blocking, waits for them to initialise, then starts the ROS2 nodes via a standard launch description.

python · swarm_launcher.py 
#!/usr/bin/env python3
"""
swarm_launcher.py
─────────────────
Launches N PX4 SITL instances + N MAVROS2 nodes, each in its own namespace.
Arms all drones and initiates the swarm takeoff sequence.

Usage: ros2 run aeroknow_swarm swarm_launcher
       (set N_DRONES=3 on machines with <8GB RAM)
"""

import rclpy
from rclpy.node  import Node
from rclpy.qos   import QoSProfile, ReliabilityPolicy, HistoryPolicy

from mavros_msgs.msg import State
from mavros_msgs.srv import CommandBool, SetMode
from geometry_msgs.msg import PoseStamped

import subprocess
import os
import time
import threading
import signal
import sys

N_DRONES      = 5
PX4_DIR       = os.path.expanduser('~/PX4-Autopilot')
WORLD_FILE    = os.path.expanduser('~/aeroknow_ws/src/aeroknow_sim/worlds/aeroknow.world')
TAKEOFF_ALT_M = 15.0   # formation cruise altitude

# Staggered spawn positions — drones start in a row (ENU x = East)
SPAWN_OFFSETS = [
    (0,   0,  0),   # drone 1 — centre
    (-5,  5,  0),   # drone 2 — 5m west, 5m north
    (-5, -5,  0),   # drone 3 — 5m west, 5m south
    (-10, 10, 0),   # drone 4 — V arm left
    (-10,-10, 0),   # drone 5 — V arm right
]

_px4_procs = []


def launch_px4_instance(instance: int):
    """Launch one PX4 SITL process."""
    env = os.environ.copy()
    env['PX4_SIM_HOST_ADDR'] = '127.0.0.1'
    env['PX4_GZ_MODEL']      = 'x500_sensors'
    env['PX4_GZ_WORLD']      = WORLD_FILE
    env['PX4_INSTANCE']      = str(instance)

    # Stagger spawn positions in the world
    ox, oy, _ = SPAWN_OFFSETS[instance]
    env['PX4_GZ_MODEL_POSE'] = f'{ox},{oy},0,0,0,0'

    cmd = ['make', '-C', PX4_DIR, f'px4_sitl', f'gz_x500_sensors_{instance}']
    proc = subprocess.Popen(cmd, env=env,
                            stdout=subprocess.DEVNULL,
                            stderr=subprocess.DEVNULL)
    _px4_procs.append(proc)
    return proc


def cleanup(signum=None, frame=None):
    """Kill all PX4 processes on exit."""
    print('\n[swarm_launcher] Shutting down all PX4 instances...')
    for p in _px4_procs:
        try: p.terminate()
        except: pass
    sys.exit(0)


class DroneArmController(Node):
    """
    Arms and takes off all N drones sequentially.
    Each drone must enter OFFBOARD mode and arm before its neighbours.
    """

    def __init__(self):
        super().__init__('swarm_arm_controller')
        rel = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                         history=HistoryPolicy.KEEP_LAST, depth=10)
        sen = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                         history=HistoryPolicy.KEEP_LAST, depth=1)

        self.states = {}   # drone_id → State msg

        # Subscribe to each drone's state and pose
        for i in range(1, N_DRONES+1):
            ns = f'/drone_{i}'
            self.create_subscription(State, f'{ns}/mavros/state',
                lambda msg, idx=i: self.states.update({idx: msg}), rel)

        # Service clients per drone
        self._arm_clis  = {i: self.create_client(CommandBool, f'/drone_{i}/mavros/cmd/arming')
                           for i in range(1, N_DRONES+1)}
        self._mode_clis = {i: self.create_client(SetMode, f'/drone_{i}/mavros/set_mode')
                           for i in range(1, N_DRONES+1)}

        # Setpoint publishers — must stream before OFFBOARD switch
        self._sp_pubs = {i: self.create_publisher(
            PoseStamped, f'/drone_{i}/mavros/setpoint_position/local', 10)
            for i in range(1, N_DRONES+1)}

        self._sp_timer = self.create_timer(0.1, self._stream_setpoints)
        self._stream_count = 0
        self._phase = 'streaming'

    def _stream_setpoints(self):
        """Publish hover setpoints to all drones continuously."""
        sp = PoseStamped()
        sp.header.stamp = self.get_clock().now().to_msg()
        sp.pose.position.z = TAKEOFF_ALT_M
        sp.pose.orientation.w = 1.0
        for pub in self._sp_pubs.values():
            pub.publish(sp)
        self._stream_count += 1

        if self._stream_count == 20 and self._phase == 'streaming':
            self._phase = 'arming'
            threading.Thread(target=self._arm_all, daemon=True).start()

    def _arm_all(self):
        """Sequentially arm all drones after streaming setpoints."""
        for i in range(1, N_DRONES+1):
            self.get_logger().info(f'Arming drone {i}...')
            # Switch to OFFBOARD
            req_mode = SetMode.Request(); req_mode.custom_mode = 'OFFBOARD'
            self._mode_clis[i].call_async(req_mode)
            time.sleep(0.5)
            # Arm
            req_arm = CommandBool.Request(); req_arm.value = True
            self._arm_clis[i].call_async(req_arm)
            time.sleep(1.0)   # stagger to avoid UDP flooding

        self.get_logger().info(f'All {N_DRONES} drones arm sequence complete. Takeoff climbing...')
        self._phase = 'flying'


def main(args=None):
    signal.signal(signal.SIGINT,  cleanup)
    signal.signal(signal.SIGTERM, cleanup)

    print(f'[swarm_launcher] Launching {N_DRONES} PX4 SITL instances...')
    for i in range(N_DRONES):
        launch_px4_instance(i)
        time.sleep(3.0)   # stagger startup — Gazebo needs time between spawns
        print(f'[swarm_launcher]   drone {i+1}/{N_DRONES} PX4 started')

    print(f'[swarm_launcher] All PX4 instances running. Waiting 10s for init...')
    time.sleep(10.0)

    rclpy.init(args=args)
    node = DroneArmController()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        cleanup()
    finally:
        node.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__': main()

Step 03: flocking_node.py — Reynolds Rules Live on a Drone

One instance of this node runs per drone. It subscribes to the positions of all swarm members, computes the four behaviour forces (separation, alignment, cohesion, formation), and publishes a velocity setpoint. This is the M09 Python simulation running on a real simulated vehicle.

STEP 03 Write flocking_node.py
Why one node instance per drone rather than one central node

A central flocking node would be a single point of failure — if it crashes, all five drones lose coordination. The decentralised approach runs one flocking node per drone. Each instance only knows about its own drone's state and its neighbours' states (via /drone_N/swarm/state topics). If one instance crashes, only that drone loses flocking — the other four continue. This is exactly the decentralised architecture from Module 09 §9.4.

#!/usr/bin/env python3
"""
flocking_node.py
────────────────
Per-drone Reynolds flocking with V-formation attraction.
One instance per drone, identified by drone_id parameter.

Subscriptions:
  /drone_{id}/mavros/local_position/pose — own position
  /drone_N/swarm/state — all neighbours' positions (for N ≠ id)

Published:
  /drone_{id}/mavros/setpoint_velocity/cmd_vel_unstamped — velocity command
  /drone_{id}/swarm/state — own state broadcast to neighbours

Reynolds weights (from Module 09 §9.4):
  separation: 3.0 — collision avoidance (highest priority)
  alignment:  1.5 — velocity matching
  cohesion:   0.8 — stay together
  formation:  2.5 — V-shape attraction
"""

import rclpy
from rclpy.node   import Node
from rclpy.qos    import QoSProfile, ReliabilityPolicy, HistoryPolicy

from geometry_msgs.msg import Twist, PoseStamped, PointStamped
from std_msgs.msg      import Float32MultiArray

import numpy as np
import math

# ── Reynolds weights ──────────────────────────────────────────────
W_SEP   = 3.0    # separation — highest priority
W_ALIGN = 1.5    # velocity alignment
W_COH   = 0.8    # cohesion to centroid
W_FORM  = 2.5    # V-formation attraction

R_SEP   = 3.0    # separation radius [m]
R_NEIGH = 20.0   # neighbourhood radius [m]
V_MAX   = 4.0    # max velocity [m/s]
ALT_KP  = 0.5    # altitude hold proportional gain
TARGET_ALT = 15.0 # cruise altitude [m]
N_DRONES   = 5

# V-formation offsets relative to swarm centroid.
# Drone 1 = tip; 2,3 = first row; 4,5 = second row.
# [x_north_m, y_east_m] offsets
FORMATION_OFFSETS = {
    1: np.array([ 0.0,   0.0]),   # leader at tip
    2: np.array([-5.0,  5.0]),   # left of leader
    3: np.array([-5.0, -5.0]),   # right of leader
    4: np.array([-10.0, 10.0]),  # outer left
    5: np.array([-10.0,-10.0]),  # outer right
}


class FlockingNode(Node):

    def __init__(self):
        super().__init__('flocking_node')

        # drone_id is set via ROS2 parameter (set in launch file)
        self.declare_parameter('drone_id', 1)
        self.drone_id = self.get_parameter('drone_id').value
        self.get_logger().info(f'FlockingNode started for drone {self.drone_id}')

        # ── Own state ────────────────────────────────────────────────
        self.own_pos = np.zeros(3)   # [x, y, z] ENU local
        self.own_vel = np.zeros(3)   # approx velocity from pos diff
        self._prev_pos = None
        self._prev_t   = None

        # ── Neighbour states ─────────────────────────────────────────
        # neighbours[drone_id] = {'pos': np.array, 'vel': np.array}
        self.neighbours = {}

        sen = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                         history=HistoryPolicy.KEEP_LAST, depth=1)
        rel = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                         history=HistoryPolicy.KEEP_LAST, depth=10)

        # ── Own position subscription ────────────────────────────────
        ns = f'/drone_{self.drone_id}'
        self.create_subscription(PoseStamped,
            f'{ns}/mavros/local_position/pose',
            self._own_pose_cb, sen)

        # ── Neighbour state subscriptions ────────────────────────────
        for j in range(1, N_DRONES+1):
            if j != self.drone_id:
                self.create_subscription(Float32MultiArray,
                    f'/drone_{j}/swarm/state',
                    lambda msg, jj=j: self._neighbour_cb(msg, jj), rel)

        # ── Publishers ───────────────────────────────────────────────
        self.vel_pub = self.create_publisher(Twist,
            f'{ns}/mavros/setpoint_velocity/cmd_vel_unstamped', 10)

        # Broadcast own state to all other flocking nodes
        self.state_pub = self.create_publisher(Float32MultiArray,
            f'{ns}/swarm/state', 10)

        # ── Control timer ────────────────────────────────────────────
        self.create_timer(0.1, self._control_loop)   # 10 Hz

    def _own_pose_cb(self, msg: PoseStamped):
        pos = np.array([msg.pose.position.x,
                        msg.pose.position.y,
                        msg.pose.position.z])
        t = self.get_clock().now().nanoseconds / 1e9

        # Finite-difference velocity estimate
        if self._prev_pos is not None and self._prev_t is not None:
            dt = t - self._prev_t
            if 0 < dt < 0.5:
                self.own_vel = (pos - self._prev_pos) / dt

        self.own_pos = pos
        self._prev_pos = pos.copy()
        self._prev_t   = t

        # Broadcast own state: [x, y, z, vx, vy, vz]
        msg_out = Float32MultiArray()
        msg_out.data = list(self.own_pos) + list(self.own_vel)
        self.state_pub.publish(msg_out)

    def _neighbour_cb(self, msg: Float32MultiArray, drone_id: int):
        if len(msg.data) >= 6:
            self.neighbours[drone_id] = {
                'pos': np.array(msg.data[:3]),
                'vel': np.array(msg.data[3:6]),
            }

    def _control_loop(self):
        """Reynolds flocking with V-formation attraction — 10 Hz."""
        sep_f  = np.zeros(2)
        align_f= np.zeros(2)
        coh_f  = np.zeros(2)
        form_f = np.zeros(2)
        n_sep  = 0; n_nei = 0

        own_offset = FORMATION_OFFSETS.get(self.drone_id, np.zeros(2))
        own_xy = self.own_pos[:2]

        for j_id, nb in self.neighbours.items():
            diff = nb['pos'][:2] - own_xy
            dist = np.linalg.norm(diff) + 1e-6

            if dist < R_SEP:
                sep_f -= diff / (dist ** 2)
                n_sep += 1

            if dist < R_NEIGH:
                align_f += nb['vel'][:2]
                coh_f   += nb['pos'][:2]
                # Formation: drive to correct relative offset from neighbour
                j_offset = FORMATION_OFFSETS.get(j_id, np.zeros(2))
                desired_diff = j_offset - own_offset
                form_f += (diff - desired_diff)
                n_nei += 1

        if n_sep > 0: sep_f /= n_sep
        if n_nei > 0:
            align_f = align_f / n_nei - self.own_vel[:2]
            coh_f   = coh_f   / n_nei - own_xy
            form_f  = form_f  / n_nei

        accel_xy = (W_SEP * sep_f + W_ALIGN * align_f +
                    W_COH * coh_f  + W_FORM  * form_f)

        # Altitude hold
        vz = ALT_KP * (TARGET_ALT - self.own_pos[2])

        # Combine and clamp speed
        vel = np.array([accel_xy[0], accel_xy[1], vz])
        speed = np.linalg.norm(vel[:2])
        if speed > V_MAX:
            vel[:2] *= V_MAX / speed

        cmd = Twist()
        cmd.linear.x = float(vel[0])
        cmd.linear.y = float(vel[1])
        cmd.linear.z = float(vel[2])
        self.vel_pub.publish(cmd)


def main(args=None):
    rclpy.init(args=args)
    node = FlockingNode()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

Step 04: cbba_allocator.py — Distributed Task Allocation

One instance per drone. Each allocator maintains a bid table and communicates bids via ROS2 topics. When the swarm receives 5 waypoint targets, the CBBA protocol converges to a conflict-free assignment in at most diam(G) communication rounds — here, with a fully-connected broadcast topology, that is 1 round.

STEP 04 Write cbba_allocator.py
The CBBA algorithm in plain English

Each drone maintains a bundle (tasks it has won) and a winning bid table (the best bid it has heard for each task from any drone). On each communication round: (1) broadcast your winning bid table to all neighbours, (2) update your table with any better bids you receive, (3) rebuild your bundle greedily — add tasks to your bundle as long as the marginal value exceeds the best competitor's bid. Convergence is guaranteed: once no drone wants to change its bundle, the assignment is final and conflict-free. Here, the reward function is simply the negative travel distance — each drone bids highest on its closest task.

#!/usr/bin/env python3
"""
cbba_allocator.py
─────────────────
Consensus-Based Bundle Algorithm — one instance per drone.
Each instance allocates tasks from a global task list using
a distributed auction. Conflict-free by design.

Topics:
  sub /cbba/tasks          (Float32MultiArray) — [lat1,lon1, lat2,lon2, ...]
  sub /cbba/bids           (Float32MultiArray) — [drone_id, task_id, bid_value, ...]
  pub /cbba/bids           (Float32MultiArray) — broadcast own bids
  pub /drone_N/cbba/assignment (std_msgs/Int32) — assigned task index (-1=none)

Reward function: -distance (closest task = highest bid)
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from std_msgs.msg      import Int32, String
from std_msgs.msg      import Float32MultiArray
from sensor_msgs.msg   import NavSatFix
from geometry_msgs.msg import PoseStamped

import numpy as np
import math
import time

N_DRONES = 5


def haversine_m(lat1, lon1, lat2, lon2) -> float:
    R = 6_371_000
    φ1, φ2 = math.radians(lat1), math.radians(lat2)
    Δφ = math.radians(lat2-lat1); Δλ = math.radians(lon2-lon1)
    a = math.sin(Δφ/2)**2 + math.cos(φ1)*math.cos(φ2)*math.sin(Δλ/2)**2
    return R*2*math.atan2(math.sqrt(a), math.sqrt(1-a))


class CBBAAllocator(Node):

    def __init__(self):
        super().__init__('cbba_allocator')

        self.declare_parameter('drone_id', 1)
        self.drone_id = self.get_parameter('drone_id').value

        # Task list — populated when /cbba/tasks is received
        self.tasks      = []      # list of (lat, lon)
        self.n_tasks    = 0

        # CBBA state tables
        self.my_bids    = {}   # task_id → my bid value
        self.best_bids  = {}   # task_id → (best_value, winner_drone_id)
        self.bundle     = []   # list of task_ids I have won
        self.converged  = False
        self._last_bundle= []

        # Own GPS position
        self.own_lat = None; self.own_lon = None

        rel = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                         history=HistoryPolicy.KEEP_LAST, depth=10)
        sen = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                         history=HistoryPolicy.KEEP_LAST, depth=1)

        ns = f'/drone_{self.drone_id}'

        self.create_subscription(Float32MultiArray, '/cbba/tasks',
            self._tasks_cb, rel)
        self.create_subscription(Float32MultiArray, '/cbba/bids',
            self._bids_cb, rel)
        self.create_subscription(NavSatFix,
            f'{ns}/mavros/global_position/global',
            self._gps_cb, sen)

        self.bids_pub      = self.create_publisher(Float32MultiArray, '/cbba/bids', 10)
        self.assignment_pub= self.create_publisher(Int32,
            f'{ns}/cbba/assignment', 10)
        self.status_pub    = self.create_publisher(String,
            f'{ns}/cbba/status', 10)

        # Auction round timer — 2 Hz
        self.create_timer(0.5, self._auction_round)

        self.get_logger().info(f'CBBAAllocator drone {self.drone_id} ready')

    def _gps_cb(self, msg: NavSatFix):
        self.own_lat = msg.latitude; self.own_lon = msg.longitude

    def _tasks_cb(self, msg: Float32MultiArray):
        """Receive task list: flat array [lat1, lon1, lat2, lon2, ...]"""
        data = msg.data
        self.tasks = [(data[i], data[i+1]) for i in range(0, len(data), 2)]
        self.n_tasks = len(self.tasks)
        # Reset CBBA state for new task set
        self.my_bids   = {j: 0.0 for j in range(self.n_tasks)}
        self.best_bids = {j: (0.0, -1) for j in range(self.n_tasks)}
        self.bundle    = []
        self.converged = False
        self.get_logger().info(
            f'Received {self.n_tasks} tasks — starting CBBA auction')

    def _bids_cb(self, msg: Float32MultiArray):
        """Receive bid broadcast: [drone_id, task_0_bid, task_1_bid, ...]"""
        if not msg.data or self.n_tasks == 0: return
        sender_id = int(msg.data[0])
        if sender_id == self.drone_id: return   # ignore own broadcasts

        bids = msg.data[1:]
        for j, bid_val in enumerate(bids[:self.n_tasks]):
            if bid_val > self.best_bids[j][0]:
                self.best_bids[j] = (bid_val, sender_id)

    def _compute_my_bid(self, task_id: int) -> float:
        """Reward = -distance to task (closer = higher bid)."""
        if self.own_lat is None or task_id >= self.n_tasks:
            return 0.0
        lat, lon = self.tasks[task_id]
        dist = haversine_m(self.own_lat, self.own_lon, lat, lon)
        return max(0.0, 1000.0 - dist)   # 1000 = max range; farther = lower bid

    def _auction_round(self):
        """One CBBA iteration: bid → broadcast → update bundle."""
        if self.n_tasks == 0 or self.own_lat is None: return

        # Step 1: compute own bids for all unallocated tasks
        for j in range(self.n_tasks):
            self.my_bids[j] = self._compute_my_bid(j)
            # Update best bid table with my own bid
            if self.my_bids[j] > self.best_bids[j][0]:
                self.best_bids[j] = (self.my_bids[j], self.drone_id)

        # Step 2: broadcast bid table
        bcast = Float32MultiArray()
        bcast.data = [float(self.drone_id)] + [self.my_bids[j] for j in range(self.n_tasks)]
        self.bids_pub.publish(bcast)

        # Step 3: build bundle — win tasks where I have the best bid
        self.bundle = [j for j in range(self.n_tasks)
                       if self.best_bids[j][1] == self.drone_id]

        # Each drone should win at most one task (1:1 assignment)
        if self.bundle:
            best_task = max(self.bundle, key=lambda j: self.my_bids[j])
            self.bundle = [best_task]
        else:
            best_task = -1

        # Step 4: check convergence (bundle unchanged for 2 rounds)
        if self.bundle == self._last_bundle and not self.converged:
            self.converged = True
            self.get_logger().info(
                f'Drone {self.drone_id}: converged → task {best_task}'
                + (f' ({self.tasks[best_task]})' if best_task >= 0 else ' (no task)')
            )
        self._last_bundle = list(self.bundle)

        # Publish assignment
        assign_msg = Int32(); assign_msg.data = int(best_task)
        self.assignment_pub.publish(assign_msg)

        status = String()
        status.data = (
            f'drone={self.drone_id} task={best_task} '
            f'bid={self.my_bids.get(best_task, 0):.1f} '
            f'converged={self.converged}'
        )
        self.status_pub.publish(status)


def main(args=None):
    rclpy.init(args=args)
    node = CBBAAllocator()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

#!/usr/bin/env python3
"""
swarm_monitor.py
────────────────
Single-terminal overview of all N drones.
Subscribes to each drone's mavros/state, local_position, battery,
and cbba/status. Refreshes display at 2 Hz.
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from mavros_msgs.msg  import State
from sensor_msgs.msg  import BatteryState
from geometry_msgs.msg import PoseStamped
from std_msgs.msg     import String, Int32

import time

N_DRONES = 5


class SwarmMonitor(Node):

    def __init__(self):
        super().__init__('swarm_monitor')

        rel = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                         history=HistoryPolicy.KEEP_LAST, depth=10)
        sen = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                         history=HistoryPolicy.KEEP_LAST, depth=1)

        # Per-drone state dict
        self.drones = {i: {
            'mode':   '—', 'armed': False,
            'alt':    0.0, 'batt':  0.0,
            'task':   -1,  'cbba':  '—',
            'x': 0.0, 'y': 0.0,
        } for i in range(1, N_DRONES+1)}

        for i in range(1, N_DRONES+1):
            ns = f'/drone_{i}'
            self.create_subscription(State,
                f'{ns}/mavros/state',
                lambda m, ii=i: self.drones[ii].update(
                    {'mode': m.mode, 'armed': m.armed}), rel)
            self.create_subscription(BatteryState,
                f'{ns}/mavros/battery',
                lambda m, ii=i: self.drones[ii].update(
                    {'batt': m.percentage*100}), sen)
            self.create_subscription(PoseStamped,
                f'{ns}/mavros/local_position/pose',
                lambda m, ii=i: self.drones[ii].update({
                    'alt': m.pose.position.z,
                    'x':   m.pose.position.x,
                    'y':   m.pose.position.y,
                }), sen)
            self.create_subscription(Int32,
                f'{ns}/cbba/assignment',
                lambda m, ii=i: self.drones[ii].update(
                    {'task': m.data}), rel)
            self.create_subscription(String,
                f'{ns}/cbba/status',
                lambda m, ii=i: self.drones[ii].update(
                    {'cbba': m.data}), rel)

        self.create_timer(0.5, self._display)
        self.t_start = time.time()
        self.get_logger().info('SwarmMonitor started')

    def _display(self):
        elapsed = time.time() - self.t_start
        print(f'\033[2J\033[H', end='')   # clear terminal
        print(f'╔{"═"*70}╗')
        print(f'║  AEROKNOW SWARM MONITOR  ·  T+{elapsed:6.1f}s  ·  {N_DRONES} drones{"":18}║')
        print(f'╠{"═"*70}╣')
        print(f'║  {"ID":<3} {"MODE":<10} {"ARM":<6} {"ALT":>6} {"BATT":>6} {"TASK":>5} {"POSITION":>18}║')
        print(f'╠{"─"*70}╣')
        for i in range(1, N_DRONES+1):
            d = self.drones[i]
            armed_s  = '\033[92mARMED \033[0m' if d['armed'] else '\033[90mDISARMD\033[0m'
            task_s   = f"WP{d['task']}"  if d['task'] >= 0 else 'NONE'
            pos_s    = f"({d['x']:+6.1f}, {d['y']:+6.1f})"
            mode_col = '\033[93m' if d['mode'] == 'OFFBOARD' else '\033[96m'
            print(f'║  {i:<3} {mode_col}{d["mode"]:<10}\033[0m {armed_s} {d["alt"]:>5.1f}m {d["batt"]:>5.1f}% {task_s:>5} {pos_s:>18} ║')
        print(f'╚{"═"*70}╝')
        # Print CBBA status for any drone that has it
        for i in range(1, N_DRONES+1):
            cbba = self.drones[i].get('cbba', '—')
            if cbba != '—':
                print(f'  D{i}: {cbba}')


def main(args=None):
    rclpy.init(args=args)
    node = SwarmMonitor()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

#!/usr/bin/env python3
"""
l08_launch.py
─────────────
ROS2 node layer for the 5-drone swarm.
PX4 processes are managed by swarm_launcher.py separately.
"""
from launch import LaunchDescription
from launch.actions import LogInfo, TimerAction
from launch_ros.actions import Node

N_DRONES = 5

# MAVROS2 FCU URLs — port offset per instance
FCU_URLS = {
    i: f'udp://:{14540+i-1}@127.0.0.1:{14557+i-1}'
    for i in range(1, N_DRONES+1)
}


def generate_launch_description():
    nodes = [LogInfo(msg=f'[AEROKNOW L08] Starting {N_DRONES}-drone swarm ROS2 stack')]

    for i in range(1, N_DRONES+1):
        ns = f'drone_{i}'

        # MAVROS2 — one per drone, unique FCU URL
        nodes.append(Node(
            package='mavros', executable='mavros_node',
            name='mavros', namespace=ns, output='log',
            parameters=[{
                'fcu_url':           FCU_URLS[i],
                'target_system_id':  i,
                'target_component_id': 1,
            }]
        ))

        # Flocking node — per drone, started after 5s (MAVROS needs time to connect)
        nodes.append(TimerAction(period=5.0, actions=[Node(
            package='aeroknow_swarm', executable='flocking_node',
            name='flocking_node', namespace=ns, output='log',
            parameters=[{'drone_id': i}]
        )]))

        # CBBA allocator — per drone
        nodes.append(TimerAction(period=5.0, actions=[Node(
            package='aeroknow_swarm', executable='cbba_allocator',
            name='cbba_allocator', namespace=ns, output='log',
            parameters=[{'drone_id': i}]
        )]))

    # Swarm monitor — single node, no namespace
    nodes.append(TimerAction(period=6.0, actions=[Node(
        package='aeroknow_swarm', executable='swarm_monitor',
        name='swarm_monitor', output='screen',
    )]))

    return LaunchDescription(nodes)

Common Errors & Fixes

Error 01 — PX4 instances start but drones occupy the same position Gazebo shows all 5 drones spawned at (0,0,0) stacked on top of each other

Cause: PX4_GZ_MODEL_POSE is not being passed to the PX4 process, or Gazebo is ignoring it for the multi-vehicle case. Fix: Verify the environment variable is set in swarm_launcher.py: print env['PX4_GZ_MODEL_POSE'] before Popen to confirm. If poses are correct but drones still stack, add an explicit Gazebo model spawn offset via the PX4_GZ_MODEL_POSE format: x,y,z,roll,pitch,yaw. Also confirm that each instance uses a unique model name — Gazebo rejects spawning a second model with the same name silently.

Error 02 — MAVROS2 nodes all connect to drone 1 All /drone_N/mavros/state topics show identical data — drone 1's state

Cause: The FCU URLs in l08_launch.py are wrong — all five MAVROS2 nodes are connecting to port 14540 (drone 1's port). Fix: Verify the FCU_URLS dict in the launch file. The listen port increments: drone 1 = :14540, drone 2 = :14541, etc. Confirm PX4 is actually listening on those ports: ss -ulnp | grep 1454 should show 5 different UDP sockets. If fewer are shown, some PX4 instances failed to start — check Terminal 1 output.

Error 03 — Flocking causes drones to collide Two drones fly toward each other and pass through (Gazebo has no collision between dynamic bodies by default)

Cause: The separation radius R_SEP = 3.0 m may be too small for the spawn separation (5 m between adjacent formation positions). If drones overshoot their formation positions, they can pass through each other before the separation force activates. Fix: Increase R_SEP to 5.0 m and reduce W_FORM from 2.5 to 1.5. This gives more separation priority relative to formation attraction. Also increase V_MAX to 6.0 if the swarm converges too slowly after startup.

Error 04 — CBBA converges but two drones get the same task D2: task=0 D4: task=0 (same task allocated to two drones)

Cause: The CBBA convergence check is seeing stale bid data — drone 4's bid for task 0 arrived before drone 2's higher bid was broadcast. This is a timing issue when the communication rate is slower than the auction round rate. Fix: Increase the auction timer period from 0.5 s to 1.0 s — giving more time for bid messages to propagate before the next round. Also confirm the /cbba/bids topic is using RELIABLE QoS — if any bid message is dropped, conflicts can result.

Error 05 — System runs out of memory, one or more PX4 instances crash Killed (signal 9) — one PX4 process terminated by OOM killer

Cause: 5 simultaneous PX4+Gazebo instances exceed available RAM. Each PX4 SITL instance uses ~400–600 MB; Gazebo with 5 sensor models uses ~1.5–2 GB. Fix: (1) Reduce to 3 drones: set N_DRONES = 3 in all files. (2) Disable the sensor model and use the lighter x500 instead of x500_sensors — remove camera and lidar plugins from the SDF. (3) Reduce Gazebo physics rate: add <real_time_factor>0.5</real_time_factor> to the world file — halves CPU/RAM at the cost of simulating at half speed.
