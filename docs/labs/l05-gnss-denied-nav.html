EKF Concepts — What robot_localization Does

Before writing a line of code, you need to understand what robot_localization is, what it expects as inputs, and what it produces. Getting the input frames and topic types wrong produces an EKF that silently runs but produces meaningless output — the most common L05 failure mode.

What robot_localization Does

robot_localization runs a 15-state EKF. The state vector is:

The 15-State Vector [x, y, z, roll, pitch, yaw, vx, vy, vz, vroll, vpitch, vyaw, ax, ay, az] — position (3), orientation (3), linear velocity (3), angular velocity (3), linear acceleration (3). All in the odom frame (ENU, origin = where node started). Not all states need sensors — states without sensor input are propagated by the IMU integration model and their covariance grows over time.

The Three Frames You Must Understand

FrameOriginUsed ForPublished By
map Fixed world origin — Earth-centred or arbitrary Absolute GPS-referenced position GPS + navsat_transform_node
odom Where the robot started (drifts relative to map over time) Local position estimate from EKF robot_localization EKF node
base_link Robot body centre Sensor transforms, attitude TF tree (MAVROS2 publishes this)

What Inputs the EKF Accepts

Message TypeMAVROS2 TopicData UsedUpdate Rate
sensor_msgs/Imu /mavros/imu/data orientation (roll/pitch/yaw), angular velocity, linear acceleration 100 Hz — drives the predict step
nav_msgs/Odometry (converted from GPS) x, y position from GPS via navsat_transform_node 5 Hz — GPS update step
nav_msgs/Odometry (from optical flow node) vx, vy velocity in body frame 30 Hz — velocity update step (replaces GPS when denied)
Why We Need navsat_transform_node GPS gives position in latitude/longitude/altitude — spherical coordinates on Earth's surface. The EKF works in Cartesian metres in the local ENU frame. navsat_transform_node (also in robot_localization) converts GPS NavSatFix messages to local Cartesian Odometry messages. You run it alongside the EKF — it needs the first GPS fix and the initial IMU heading to establish the local frame origin. Once GPS is denied, the node stops producing odometry and the EKF continues on IMU + optical flow only.

Step 01: Install Dependencies and Create TF Publisher

STEP 01 Install packages and write the TF static broadcaster
Why a TF broadcaster is needed

The EKF needs a complete TF tree to know how sensors relate to the drone body. MAVROS2 publishes the base_link → odom transform, but it does not publish the sensor-to-body transforms (camera position relative to the drone centre). Without these, robot_localization cannot correctly account for the lever arm offset between the IMU and the camera — the exact source of error analysed in Module 05 §5.5. We publish static transforms that match the positions defined in our model.sdf from L04.

#!/usr/bin/env python3
"""
static_tf_publisher.py
──────────────────────
Publishes static TF transforms for all sensor links relative to base_link.
Values match the <pose> entries in the L04 model.sdf exactly.

These transforms tell robot_localization where each sensor
physically is relative to the drone's centre of mass —
this is the lever arm correction from Module 05 §5.5.
"""
import rclpy
from rclpy.node import Node
from tf2_ros import StaticTransformBroadcaster
from geometry_msgs.msg import TransformStamped
import math


def make_tf(parent, child, x, y, z, roll=0, pitch=0, yaw=0) -> TransformStamped:
    """Build a static transform message."""
    t = TransformStamped()
    t.header.frame_id = parent
    t.child_frame_id  = child
    t.transform.translation.x = float(x)
    t.transform.translation.y = float(y)
    t.transform.translation.z = float(z)

    # Convert Euler angles to quaternion
    cr, sr = math.cos(roll/2),  math.sin(roll/2)
    cp, sp = math.cos(pitch/2), math.sin(pitch/2)
    cy, sy = math.cos(yaw/2),  math.sin(yaw/2)
    t.transform.rotation.w = cr*cp*cy + sr*sp*sy
    t.transform.rotation.x = sr*cp*cy - cr*sp*sy
    t.transform.rotation.y = cr*sp*cy + sr*cp*sy
    t.transform.rotation.z = cr*cp*sy - sr*sp*cy
    return t


class StaticTFPublisher(Node):

    def __init__(self):
        super().__init__('static_tf_publisher')
        br = StaticTransformBroadcaster(self)
        now = self.get_clock().now().to_msg()

        transforms = []

        # base_link → camera_link
        # Matches model.sdf: 0.05m forward, 0m lateral, 0.06m below, pitch 90° down
        tf_cam = make_tf('base_link', 'camera_link',
                         x=0.05, y=0.0, z=-0.06,
                         pitch=math.pi/2)
        tf_cam.header.stamp = now
        transforms.append(tf_cam)

        # base_link → depth_link
        # Matches model.sdf: 0.12m forward, level
        tf_dep = make_tf('base_link', 'depth_link',
                         x=0.12, y=0.0, z=0.0)
        tf_dep.header.stamp = now
        transforms.append(tf_dep)

        # base_link → lidar_link
        # Matches model.sdf: 0.1m above body
        tf_lid = make_tf('base_link', 'lidar_link',
                         x=0.0, y=0.0, z=0.1)
        tf_lid.header.stamp = now
        transforms.append(tf_lid)

        # map → odom (identity — same frame at startup)
        # robot_localization will update this continuously
        tf_map = make_tf('map', 'odom', 0, 0, 0)
        tf_map.header.stamp = now
        transforms.append(tf_map)

        br.sendTransform(transforms)
        self.get_logger().info('Static TF frames published: camera_link, depth_link, lidar_link')


def main(args=None):
    rclpy.init(args=args)
    node = StaticTFPublisher()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__': main()

Step 02–03: EKF Configuration YAML

robot_localization is configured entirely through a YAML parameter file. Every number in this file has a physical meaning grounded in the sensor noise models from Module 05. Getting this wrong is the most common cause of EKF divergence.

STEP 02 Write ekf_params.yaml
How to read the YAML config

The imu0_config array is 15 booleans corresponding to the 15 EKF states: [x,y,z, roll,pitch,yaw, vx,vy,vz, vroll,vpitch,vyaw, ax,ay,az]. true = "use this measurement from this sensor to update this state." false = "ignore." The IMU provides orientation and angular rates — not raw position. The odometry (from GPS conversion) provides x and y position. Setting the wrong states to true causes conflicting updates that make the filter diverge.

Step 03–04: ekf_nav_node.py

The EKF nav node does three things: launches and manages the robot_localization EKF via a subprocess, computes a simple optical flow velocity estimate from consecutive camera frames (the GPS-denied velocity source), and monitors when GPS transitions from present to absent so it can log the event and alert the health monitor.

STEP 03 Write ekf_nav_node.py
Why compute optical flow here instead of a separate node

Optical flow for ego-motion (how fast the drone is moving over the ground) only needs two consecutive frames and the camera altitude — it's a simple Lucas-Kanade sparse flow computation, not a full RAFT dense flow network (Module 07). We compute it here because it is tightly coupled to the EKF timing: the flow velocity estimate must be published as a nav_msgs/Odometry with a covariance that reflects how well the flow could be estimated (low texture → high covariance, high texture → low covariance). Coupling the computation and the covariance assignment in one node prevents timing mismatches.

#!/usr/bin/env python3
"""
ekf_nav_node.py
───────────────
Manages the GNSS-denied navigation stack:
  1. Converts camera frames to sparse optical flow velocity
  2. Publishes flow velocity as nav_msgs/Odometry for robot_localization
  3. Monitors GPS fix status and logs denial events
  4. Adjusts optical flow covariance based on feature count

Topics published:
  /odometry/optical_flow  (nav_msgs/Odometry) — flow-based velocity
  /nav/gps_denied        (std_msgs/Bool)       — True when GPS is denied

Topics subscribed:
  /camera/image_raw       — for optical flow computation
  /mavros/global_position/global — to detect GPS denial
  /mavros/local_position/pose    — altitude for flow scaling
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from sensor_msgs.msg  import Image, NavSatFix
from geometry_msgs.msg import PoseStamped
from nav_msgs.msg     import Odometry
from std_msgs.msg     import Bool

import numpy as np
import cv2
import time

try:
    from cv_bridge import CvBridge
    CV_BRIDGE_OK = True
except ImportError:
    CV_BRIDGE_OK = False

# Optical flow parameters
LK_PARAMS = dict(
    winSize   = (21, 21),
    maxLevel  = 3,
    criteria  = (cv2.TERM_CRITERIA_EPS | cv2.TERM_CRITERIA_COUNT, 30, 0.01),
)
FEATURE_PARAMS = dict(
    maxCorners  = 100,
    qualityLevel= 0.3,
    minDistance = 7,
    blockSize   = 7,
)
MIN_FEATURES = 10   # below this, flow is unreliable — set high covariance
GPS_STALE_S  = 2.0   # seconds without GPS before declaring denial


class EKFNavNode(Node):

    def __init__(self):
        super().__init__('ekf_nav_node')

        # ── State ────────────────────────────────────────────────────
        self.prev_gray    = None       # previous camera frame (grayscale)
        self.prev_pts     = None       # tracked feature points in prev frame
        self.prev_t       = None       # timestamp of previous frame
        self.altitude_m   = 1.0        # current altitude AGL [m] — for flow scaling
        self.last_gps_t   = time.time() # last time GPS message was received
        self.gps_denied   = False
        self.bridge       = CvBridge() if CV_BRIDGE_OK else None

        sen_qos = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                             history=HistoryPolicy.KEEP_LAST, depth=1)
        rel_qos = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                             history=HistoryPolicy.KEEP_LAST, depth=10)

        # ── Subscriptions ────────────────────────────────────────────
        self.create_subscription(Image,    '/camera/image_raw',
            self.image_cb, sen_qos)
        self.create_subscription(NavSatFix, '/mavros/global_position/global',
            self.gps_cb, sen_qos)
        self.create_subscription(PoseStamped, '/mavros/local_position/pose',
            self.pose_cb, sen_qos)

        # ── Publishers ───────────────────────────────────────────────
        self.flow_pub   = self.create_publisher(Odometry, '/odometry/optical_flow', 10)
        self.denied_pub = self.create_publisher(Bool, '/nav/gps_denied', 10)

        # ── GPS denial watchdog ──────────────────────────────────────
        # Fires every 0.5s — checks if GPS has gone stale
        self.create_timer(0.5, self.gps_watchdog)

        self.get_logger().info('EKFNavNode started')
        if not CV_BRIDGE_OK:
            self.get_logger().warn('cv_bridge not installed — optical flow disabled')

    # ── Callbacks ────────────────────────────────────────────────────

    def gps_cb(self, msg: NavSatFix):
        """Record the time of the last valid GPS message."""
        if msg.status.status >= 0:   # -1 = no fix, 0+ = fix
            self.last_gps_t = time.time()

    def pose_cb(self, msg: PoseStamped):
        """Track altitude for optical flow pixel-to-metre conversion."""
        self.altitude_m = max(0.5, msg.pose.position.z)

    def gps_watchdog(self):
        """Detect GPS denial (no valid GPS for GPS_STALE_S seconds)."""
        was_denied = self.gps_denied
        self.gps_denied = (time.time() - self.last_gps_t) > GPS_STALE_S

        if self.gps_denied and not was_denied:
            self.get_logger().warn(
                'GPS DENIED — switching to IMU + optical flow navigation')
        elif not self.gps_denied and was_denied:
            self.get_logger().info('GPS RESTORED')

        msg = Bool(); msg.data = self.gps_denied
        self.denied_pub.publish(msg)

    def image_cb(self, msg: Image):
        """
        Compute Lucas-Kanade sparse optical flow between consecutive frames.
        Convert pixel displacement to ground velocity using altitude.
        Publish as nav_msgs/Odometry for robot_localization.
        """
        if not CV_BRIDGE_OK or self.bridge is None:
            return

        # Convert ROS2 Image → OpenCV numpy array
        try:
            frame = self.bridge.imgmsg_to_cv2(msg, desired_encoding='bgr8')
        except Exception as e:
            self.get_logger().warn(f'Image conversion failed: {e}')
            return

        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        t_now = time.time()

        if self.prev_gray is None:
            # First frame — initialise
            self.prev_gray = gray
            self.prev_pts  = cv2.goodFeaturesToTrack(gray, mask=None, **FEATURE_PARAMS)
            self.prev_t    = t_now
            return

        dt = t_now - self.prev_t
        if dt <= 0 or dt > 1.0:   # skip if dt is unreasonable
            self.prev_gray = gray
            self.prev_t    = t_now
            return

        # Track existing feature points into the new frame
        vx, vy = 0.0, 0.0
        n_good  = 0
        covariance_scale = 1.0   # multiplier on velocity covariance (1 = nominal)

        if self.prev_pts is not None and len(self.prev_pts) >= MIN_FEATURES:
            curr_pts, status, _ = cv2.calcOpticalFlowPyrLK(
                self.prev_gray, gray, self.prev_pts, None, **LK_PARAMS)

            if curr_pts is not None and status is not None:
                good_prev = self.prev_pts[status == 1]
                good_curr = curr_pts[status == 1]
                n_good = len(good_curr)

                if n_good >= MIN_FEATURES:
                    # Mean pixel displacement per second
                    displacement = (good_curr - good_prev).reshape(-1, 2)
                    mean_dx = np.median(displacement[:, 0]) / dt  # px/s
                    mean_dy = np.median(displacement[:, 1]) / dt  # px/s

                    # Convert px/s → m/s using altitude and camera FOV
                    # FOV = 80° (from model.sdf), image width = 640px
                    # angular_rate [rad/s] = displacement_px/s × (FOV_rad / image_width_px)
                    # linear_velocity [m/s] = angular_rate × altitude
                    fov_rad   = 1.3963   # 80° in radians
                    img_width = 640.0
                    px_to_rad = fov_rad / img_width
                    # Camera faces down: x-pixel movement = eastward motion, y-pixel = southward
                    vx = -mean_dx * px_to_rad * self.altitude_m  # positive East
                    vy = -mean_dy * px_to_rad * self.altitude_m  # positive North

                    # Better features → lower covariance (more confidence)
                    covariance_scale = max(0.1, MIN_FEATURES / n_good)
        else:
            # Too few features — high covariance (don't trust this measurement much)
            covariance_scale = 10.0
            self.get_logger().warn(
                f'Low features: {len(self.prev_pts) if self.prev_pts is not None else 0}'
                ' — optical flow unreliable',
                throttle_duration_sec=2.0
            )

        # Refresh feature set every 10 frames to avoid drift in tracking
        if n_good < MIN_FEATURES * 2:
            self.prev_pts = cv2.goodFeaturesToTrack(gray, mask=None, **FEATURE_PARAMS)
        else:
            self.prev_pts = good_curr.reshape(-1, 1, 2)

        self.prev_gray = gray
        self.prev_t    = t_now

        # ── Publish as Odometry for robot_localization ───────────────
        # Only vx and vy are set (odom1_config uses only vx, vy).
        # The covariance diagonal entries [0][0] and [1][1] are for vx, vy.
        # Higher covariance_scale = less confidence in this measurement.
        odom = Odometry()
        odom.header.stamp    = self.get_clock().now().to_msg()
        odom.header.frame_id = 'odom'
        odom.child_frame_id  = 'base_link'
        odom.twist.twist.linear.x = float(vx)
        odom.twist.twist.linear.y = float(vy)

        # 6×6 covariance for twist (velocity): [vx,vy,vz,vroll,vpitch,vyaw]
        # Only vx, vy entries matter since odom1_config only uses those.
        base_vel_cov = 0.15 * covariance_scale   # m²/s²
        odom.twist.covariance[0]  = base_vel_cov   # vx variance
        odom.twist.covariance[7]  = base_vel_cov   # vy variance
        odom.twist.covariance[14] = 9999.0         # vz — not observable from downward flow
        odom.twist.covariance[21] = 9999.0
        odom.twist.covariance[28] = 9999.0
        odom.twist.covariance[35] = 9999.0

        self.flow_pub.publish(odom)


def main(args=None):
    rclpy.init(args=args)
    node = EKFNavNode()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass

Step 05: gps_denial_injector.py

You cannot physically jam GPS inside your computer. Instead, the denial injector intercepts the GPS topic and stops forwarding messages — from the EKF's perspective, this is indistinguishable from a real jammer killing the signal. It can be triggered by time, by waypoint entry, or by an external command.

STEP 05 Write gps_denial_injector.py
Why intercept and re-publish rather than mute the topic

ROS2 has no built-in way to "mute" a topic at runtime. You cannot unsubscribe a remote publisher. The correct approach is to create a relay: subscribe to the real GPS topic and republish on a new topic (/mavros/global_position/global_relayed). When denial is active, simply stop republishing. The EKF subscribes to the relayed topic, not the raw one. This pattern — intercept, transform/filter, republish — is the standard ROS2 approach to runtime topic modification.

bash 
$ nano ~/aeroknow_ws/src/aeroknow_bringup/aeroknow_bringup/gps_denial_injector.py
python · gps_denial_injector.py 
#!/usr/bin/env python3
"""
gps_denial_injector.py
──────────────────────
Relays GPS topic — stops relaying when denial is active.
From the EKF's perspective, GPS simply stops arriving.

Relay chain:
  /mavros/global_position/global
      ↓ (relayed when not denied)
  /mavros/global_position/global_relayed
      ↓ (navsat_transform_node subscribes here)
  /odometry/gps
      ↓ (EKF odom0 input)

Trigger:
  # Deny GPS now:
  ros2 topic pub --once /gps_denial/command std_msgs/msg/Bool "data: true"
  # Restore GPS:
  ros2 topic pub --once /gps_denial/command std_msgs/msg/Bool "data: false"

  # Deny for 30 seconds starting in 5 seconds:
  ros2 topic pub --once /gps_denial/timed std_msgs/msg/String "data: '5 30'"
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from sensor_msgs.msg import NavSatFix
from std_msgs.msg    import Bool, String

import time


class GPSDenialInjector(Node):

    def __init__(self):
        super().__init__('gps_denial_injector')

        self.denied      = False
        self.deny_until  = 0.0   # epoch time to re-enable GPS
        self.n_blocked   = 0     # count of blocked GPS messages
        self.n_passed    = 0     # count of passed GPS messages

        sen_qos = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                             history=HistoryPolicy.KEEP_LAST, depth=1)
        rel_qos = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                             history=HistoryPolicy.KEEP_LAST, depth=10)

        # Subscribe to real GPS
        self.create_subscription(NavSatFix,
            '/mavros/global_position/global',
            self.gps_relay_cb, sen_qos)

        # Control topics
        self.create_subscription(Bool,
            '/gps_denial/command', self.command_cb, rel_qos)
        self.create_subscription(String,
            '/gps_denial/timed', self.timed_cb, rel_qos)

        # Publish relayed GPS (navsat_transform_node subscribes here)
        self.gps_pub = self.create_publisher(NavSatFix,
            '/mavros/global_position/global_relayed', 10)

        # Status publisher (nav_health_monitor listens)
        self.status_pub = self.create_publisher(Bool, '/gps_denial/active', 10)

        # Stats timer
        self.create_timer(5.0, self.print_stats)

        self.get_logger().info(
            'GPSDenialInjector active.\n'
            '  Deny:    ros2 topic pub --once /gps_denial/command std_msgs/msg/Bool "data: true"\n'
            '  Restore: ros2 topic pub --once /gps_denial/command std_msgs/msg/Bool "data: false"\n'
            '  Timed:   ros2 topic pub --once /gps_denial/timed std_msgs/msg/String "data: \'5 30\'"'
        )

    def gps_relay_cb(self, msg: NavSatFix):
        # Check if a timed denial has expired
        if self.deny_until > 0 and time.time() > self.deny_until:
            self.denied     = False
            self.deny_until = 0.0
            self.get_logger().info('Timed GPS denial expired — restoring GPS')

        if self.denied:
            self.n_blocked += 1
            # Do NOT republish — GPS simply stops appearing to the EKF
        else:
            self.n_passed += 1
            self.gps_pub.publish(msg)

        status = Bool(); status.data = self.denied
        self.status_pub.publish(status)

    def command_cb(self, msg: Bool):
        prev = self.denied
        self.denied = msg.data
        if self.denied and not prev:
            self.get_logger().warn('GPS DENIAL ACTIVATED — blocking GPS relay')
        elif not self.denied and prev:
            self.get_logger().info('GPS DENIAL DEACTIVATED — resuming relay')

    def timed_cb(self, msg: String):
        """Parse 'delay_s duration_s' and schedule a timed denial."""
        parts = msg.data.strip().split()
        if len(parts) != 2:
            self.get_logger().warn(f'timed: expected "delay_s duration_s", got: {msg.data}')
            return
        delay_s    = float(parts[0])
        duration_s = float(parts[1])
        start_t    = time.time() + delay_s
        self.deny_until = start_t + duration_s

        # Schedule the start
        self.create_timer(delay_s, self._start_timed_denial)
        self.get_logger().info(
            f'Timed GPS denial scheduled: starts in {delay_s:.0f}s, '
            f'lasts {duration_s:.0f}s'
        )

    def _start_timed_denial(self):
        self.denied = True
        self.get_logger().warn('Timed GPS denial STARTED')

    def print_stats(self):
        total = self.n_passed + self.n_blocked
        if total > 0:
            pct = self.n_blocked / total * 100
            self.get_logger().info(
                f'GPS relay stats: {self.n_passed} passed, {self.n_blocked} blocked ({pct:.0f}%)',
                throttle_duration_sec=5.0
            )


def main(args=None):
    rclpy.init(args=args)
    node = GPSDenialInjector()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

Step 06: nav_health_monitor.py

The EKF outputs a covariance matrix alongside every position estimate. As GPS denial continues and only IMU + optical flow are available, the covariance grows — the filter knows its estimate is becoming less certain. The health monitor watches this covariance and triggers an RTB if it exceeds a safe threshold. This is the chi-square RAIM concept from Module 05 §5.6 applied to the navigation output.

STEP 06 Write nav_health_monitor.py
Why watch covariance, not just position error

In a real scenario, you don't know the true position — there's no ground truth to compare against. The covariance is what the EKF itself believes about its own uncertainty. When the covariance position diagonal entries exceed a threshold (e.g., σ_x > 5 m), the filter is telling you: "I am no longer confident in my position estimate to within a safe margin." This is the operationally correct trigger for RTB — you act on the filter's self-assessed uncertainty, not on a comparison to external ground truth you don't have.

bash 
$ nano ~/aeroknow_ws/src/aeroknow_bringup/aeroknow_bringup/nav_health_monitor.py
python · nav_health_monitor.py 
#!/usr/bin/env python3
"""
nav_health_monitor.py
─────────────────────
Watches the robot_localization EKF covariance matrix.
Classifies navigation health and triggers RTB when covariance
exceeds safe operational limits.

Covariance thresholds (Module 05 §5.6, chi-square test context):
  σ_xy < 2.0 m → NOMINAL
  σ_xy < 5.0 m → DEGRADED (warn, continue)
  σ_xy ≥ 5.0 m → CRITICAL  (trigger RTB)

Subscribed topics:
  /odometry/filtered (nav_msgs/Odometry) — EKF output with covariance
  /nav/gps_denied    (std_msgs/Bool)      — GPS denial status

Published topics:
  /nav_health        (std_msgs/String)    — NOMINAL / DEGRADED / CRITICAL
  /emergency_stop    (std_msgs/Bool)      — True if CRITICAL and RTB triggered
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from nav_msgs.msg  import Odometry
from std_msgs.msg  import Bool, String

import math

SIGMA_NOMINAL   = 2.0   # [m] max σ_xy for NOMINAL
SIGMA_DEGRADED  = 5.0   # [m] max σ_xy before RTB
RTB_HOLD_S      = 3.0   # [s] must be CRITICAL for this long before RTB fires


class NavHealthMonitor(Node):

    def __init__(self):
        super().__init__('nav_health_monitor')

        self.sigma_xy    = 0.0
        self.sigma_z     = 0.0
        self.gps_denied  = False
        self.health      = 'NOMINAL'
        self.critical_since = None   # time CRITICAL state started
        self.rtb_fired   = False

        sen_qos = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                             history=HistoryPolicy.KEEP_LAST, depth=1)
        rel_qos = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                             history=HistoryPolicy.KEEP_LAST, depth=10)

        self.create_subscription(Odometry, '/odometry/filtered',
            self.odom_cb, sen_qos)
        self.create_subscription(Bool, '/nav/gps_denied',
            self.denial_cb, rel_qos)

        self.health_pub = self.create_publisher(String, '/nav_health', 10)
        self.estop_pub  = self.create_publisher(Bool, '/emergency_stop', 10)

        # Display timer
        self.create_timer(1.0, self.display)
        self.get_logger().info('NavHealthMonitor started')

    def denial_cb(self, msg: Bool):
        self.gps_denied = msg.data

    def odom_cb(self, msg: Odometry):
        # Extract position standard deviations from the covariance matrix.
        # nav_msgs/Odometry pose covariance is a 6×6 matrix (36 elements).
        # Elements [0] = xx, [7] = yy, [14] = zz (diagonal, 0-indexed).
        # σ = sqrt(variance)
        cov = msg.pose.covariance
        self.sigma_xy = math.sqrt(max(0, (cov[0] + cov[7]) / 2))  # RMS of x and y σ
        self.sigma_z  = math.sqrt(max(0, cov[14]))

        # Classify health
        prev_health = self.health
        if self.sigma_xy < SIGMA_NOMINAL:
            self.health = 'NOMINAL'
            self.critical_since = None
        elif self.sigma_xy < SIGMA_DEGRADED:
            self.health = 'DEGRADED'
            self.critical_since = None
        else:
            self.health = 'CRITICAL'

        if self.health != prev_health:
            self.get_logger().warn(
                f'Nav health: {prev_health} → {self.health} '
                f'(σ_xy={self.sigma_xy:.2f}m, GPS_denied={self.gps_denied})'
            )

        # Track how long we've been CRITICAL
        import time
        if self.health == 'CRITICAL':
            if self.critical_since is None:
                self.critical_since = time.time()
            elif time.time() - self.critical_since >= RTB_HOLD_S and not self.rtb_fired:
                self.rtb_fired = True
                self.get_logger().error(
                    f'NAV CRITICAL for {RTB_HOLD_S}s — triggering RTB/emergency stop! '
                    f'σ_xy={self.sigma_xy:.2f}m'
                )
                stop_msg = Bool(); stop_msg.data = True
                self.estop_pub.publish(stop_msg)
        else:
            self.rtb_fired = False   # reset if health improves

        # Publish health string
        h_msg = String(); h_msg.data = (
            f'{self.health}|σ_xy={self.sigma_xy:.3f}m|σ_z={self.sigma_z:.3f}m'
            f'|GPS_denied={self.gps_denied}'
        )
        self.health_pub.publish(h_msg)

    def display(self):
        colour = ('\033[92m' if self.health == 'NOMINAL'
                  else '\033[93m' if self.health == 'DEGRADED'
                  else '\033[91m')
        self.get_logger().info(
            f'{colour}{self.health}\033[0m | '
            f'σ_xy={self.sigma_xy:.3f}m | σ_z={self.sigma_z:.3f}m | '
            f'GPS denied={self.gps_denied}',
            throttle_duration_sec=1.0
        )


def main(args=None):
    rclpy.init(args=args)
    node = NavHealthMonitor()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

Common Errors & Fixes

Error 01 — EKF diverges immediately after start ekf_filter_node: covariance matrix is not positive definite

Cause: One of the sensor inputs has a covariance set to zero, which is numerically invalid for the EKF. This usually means a MAVROS2 topic is publishing with an all-zero covariance field — some firmware versions do this for GPS before a fix is acquired. Fix: Add a small minimum covariance floor in the YAML — raise odom0_pose_rejection_threshold to 15.0 temporarily. Also confirm navsat_transform_node is receiving a valid first GPS fix (check ros2 topic echo /mavros/global_position/global — status field must be ≥ 0). The EKF cannot initialise without at least one valid GPS message.

Error 02 — /odometry/filtered not published ros2 topic list → /odometry/filtered absent

Cause: ekf_filter_node did not receive any valid sensor input within the sensor_timeout period, so it never started publishing. Common sub-causes: (1) TF tree incomplete — static_tf_publisher not running, (2) IMU topic QoS mismatch, (3) YAML parameter file not found by the node. Fix: (1) Run ros2 run tf2_tools view_frames — confirm base_link → odom exists. (2) Check IMU: ros2 topic hz /mavros/imu/data should show ~100 Hz. (3) Confirm the YAML path in the launch file matches ros2 run aeroknow_bringup --ros-args --params-file /path/to/ekf_params.yaml.

Error 03 — Optical flow publishes zero velocity always ekf_nav_node: Low features: 0 — optical flow unreliable (constant)

Cause: Either cv_bridge is not installed, or the camera topic is not publishing, or the image is all black (rendering issue from L04 Error 03). Fix: (1) pip3 install cv-bridge or sudo apt install ros-humble-cv-bridge. (2) Check camera: ros2 topic hz /camera/image_raw — should show ~30 Hz. (3) If image is black, re-apply the L04 fix for software rendering. Optical flow is not critical for the EKF to run — it degrades gracefully without it, the covariance just grows faster during denial.

Error 04 — navsat_transform_node never publishes /odometry/gps ros2 topic hz /odometry/gps → 0 Hz (not publishing)

Cause: navsat_transform_node needs a valid initial IMU orientation and GPS fix to establish the local frame origin. It waits up to delay: 3.0 seconds before giving up on the first fix. Fix: Confirm the remapping in the launch file is correct — gps/fix must map to /mavros/global_position/global_relayed (the injector's output), not the raw GPS topic. Run ros2 topic echo /mavros/global_position/global_relayed — if this is empty, the injector is not running or is in denial mode.

Error 05 — RTB triggers immediately on first EKF start [ERROR] NAV CRITICAL for 3s — triggering RTB! σ_xy=999.99m

Cause: The initial EKF covariance before any sensor updates is very large (the prior is uncertain). The health monitor sees this and triggers RTB before the EKF has had time to converge. Fix: Add a startup delay to the health monitor — only start evaluating health after 10 seconds (let the EKF converge first). Add self.startup_done = False in __init__ and self.create_timer(10.0, lambda: setattr(self, 'startup_done', True)). Then gate the RTB trigger: if not self.startup_done: return at the top of odom_cb.
