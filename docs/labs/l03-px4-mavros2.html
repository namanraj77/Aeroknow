⟐ Theory Link

L03 is the practical implementation of the cascade control loop from Module 03 — GNC Fundamentals §3.5. The flight controller node operates at the outer (position) loop level; MAVROS2 translates its setpoints to the inner PX4 attitude and rate loops. Understanding the bandwidth separation principle (§3.5) explains why setpoints must be published at a minimum rate — violating it causes Offboard mode rejection.

PX4 Flight Modes — What They Are and Why They Matter

PX4 is always in a flight mode. The mode determines who is in control and what kind of commands PX4 accepts. Getting the mode wrong is the number one reason Python commands appear to do nothing. You must understand modes before writing any flight code.

Mode Who Controls Accepts From ROS2? Used In
MANUAL RC transmitter only No Manual override, emergency
STABILIZED RC + attitude stabilisation No Manual flying with assist
POSITION GPS hold + RC No Hover at fixed GPS point
OFFBOARD External computer (your ROS2 node) Yes — setpoints via MAVROS2 topics All autonomous flight in L03–L08
AUTO.MISSION PX4 internal mission planner Via mission upload then trigger Pre-uploaded waypoint missions
AUTO.LAND PX4 automatic landing Triggered via /mavros/cmd/land Normal landing, emergency stop
AUTO.RTL PX4 returns to launch point Triggered via /mavros/cmd/set_home RTB on comms loss

The Offboard Mode Catch

OFFBOARD mode has one critical rule that catches every beginner:

Offboard Mode Requires Setpoints Before Mode Switch PX4 will refuse to enter OFFBOARD mode unless it has been receiving valid setpoint messages at ≥ 2 Hz for at least 0.5 seconds before the mode switch command is sent. If you call set_mode("OFFBOARD") without first streaming setpoints, PX4 silently rejects the mode change and stays in the previous mode — with no error message in the terminal. The fix is always: start the setpoint stream first, wait 1 second, then switch mode. Every node in L03 follows this exact pattern.

The Arming Sequence

Even in OFFBOARD mode, the motors won't spin until the drone is armed. Arming means "I intend to fly — motors may start." Disarming is the first safety action in any emergency. The sequence for autonomous flight is always:

PHASE 1
Stream setpoints
Publish setpoints at 10+ Hz for 1+ seconds before doing anything else
PHASE 2
Switch to OFFBOARD
Call set_mode("OFFBOARD") via MAVROS2 service
PHASE 3
Arm motors
Call arming(True) via MAVROS2 service
PHASE 4
Publish climb setpoint
Publish target altitude; keep streaming at 10+ Hz while drone climbs
PHASE 5
Execute mission
Publish sequential waypoint setpoints; monitor distance to each
PHASE 6
Land
Call land() service — PX4 handles the descent and touchdown

Offboard Control — The Three Setpoint Types

In OFFBOARD mode, MAVROS2 exposes three ways to command the drone. Each one commands a different level of the PX4 cascade controller. Picking the right one for the task is an engineering decision, not a preference.

Setpoint TypeMAVROS2 TopicMessage TypePX4 LevelBest For
Position /mavros/setpoint_position/local geometry_msgs/PoseStamped Outer loop (guidance) Waypoint navigation — "go to this exact XYZ coordinate"
Velocity /mavros/setpoint_velocity/cmd_vel_unstamped geometry_msgs/Twist Mid loop (velocity) Following paths, obstacle avoidance — "fly at this speed in this direction"
Attitude /mavros/setpoint_attitude/attitude geometry_msgs/PoseStamped + thrust Inner loop Aerobatics, precise attitude control — use with extreme care

In L03, you will use position setpoints for the main mission (go to GPS waypoints) and velocity setpoints for the square-path demo (maintain a specific speed and direction). L05 adds a velocity-mode EKF correction layer on top of the same setpoint interface.

Local vs. Global Coordinates MAVROS2 has two position setpoint topics. /mavros/setpoint_position/local takes NED coordinates (North-East-Down) in metres relative to the home position — this is what we use in L03. /mavros/setpoint_position/global takes GPS latitude/longitude/altitude — this is what the L02 action server used. Local coordinates are easier for precise geometric missions (fly a 20m square); global coordinates are needed when working with maps and real-world positions.

Step 04: flight_controller.py

The core node. Arms, takes off, flies to three waypoints in sequence, then lands. This is the first time your code physically moves something in the simulation.

STEP 04 Write flight_controller.py
Why this code is structured this way

The controller runs as a state machine inside a timer callback — not a while True loop. This matters because a loop would block all ROS2 callbacks (including the position subscriber that tells us when we've arrived at a waypoint). The timer fires every 0.1 s (10 Hz), checks the current state, and publishes the appropriate setpoint. The state advances when the condition for the next state is met — for example, the drone is within 0.5 m of the target altitude before arming transitions to flying.

#!/usr/bin/env python3
"""
flight_controller.py
────────────────────
Autonomous arm → takeoff → 3 waypoints → land mission.
Uses position setpoints in local NED frame.

State machine:
  IDLE → STREAMING → OFFBOARD → ARMING → TAKEOFF
       → WP_0 → WP_1 → WP_2 → LANDING → DONE
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from geometry_msgs.msg  import PoseStamped
from mavros_msgs.msg    import State
from mavros_msgs.srv    import CommandBool, SetMode
from std_srvs.srv       import Trigger

import math


# Waypoints: [x_north_m, y_east_m, z_up_m] in local NED
# NED convention: Z positive = DOWN in raw MAVLink,
# but MAVROS2 setpoint_position/local uses ENU (Z positive = UP).
# So z=20 means 20 metres above home. Positive x = North, positive y = East.
WAYPOINTS = [
    ( 0.0,   0.0,  20.0),   # WP0: takeoff — hover at 20 m directly above home
    (20.0,   0.0,  20.0),   # WP1: fly 20 m north
    (20.0,  20.0,  20.0),   # WP2: fly 20 m east
    ( 0.0,  20.0,  20.0),   # WP3: return north (forms an L-shape)
]

ACCEPTANCE_RADIUS_M = 1.5   # within this distance → waypoint reached
SETPOINT_RATE_HZ    = 10.0  # must be ≥ 2 Hz for Offboard; 10 Hz is safe


class FlightController(Node):

    def __init__(self):
        super().__init__('flight_controller')

        # ── State machine ────────────────────────────────────────────
        self.state      = 'IDLE'
        self.wp_index   = 0       # current waypoint index
        self.stream_count = 0    # how many setpoints we've sent before mode switch

        # ── Current vehicle state (from MAVROS2) ─────────────────────
        self.fcu_mode   = ''
        self.armed      = False
        self.connected  = False
        self.local_x    = 0.0
        self.local_y    = 0.0
        self.local_z    = 0.0

        # ── QoS ──────────────────────────────────────────────────────
        rel_qos = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                             history=HistoryPolicy.KEEP_LAST, depth=10)
        sen_qos = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                             history=HistoryPolicy.KEEP_LAST, depth=1)

        # ── Subscribers ──────────────────────────────────────────────
        self.create_subscription(State, '/mavros/state',
            self.state_cb, rel_qos)
        self.create_subscription(PoseStamped, '/mavros/local_position/pose',
            self.pose_cb, sen_qos)

        # ── Publisher: position setpoints ────────────────────────────
        self.sp_pub = self.create_publisher(
            PoseStamped, '/mavros/setpoint_position/local', 10)

        # ── Service clients ──────────────────────────────────────────
        # We create these but don't call them until the correct state.
        # wait_for_service() ensures MAVROS2 is ready before we try.
        self.set_mode_cli = self.create_client(SetMode, '/mavros/set_mode')
        self.arming_cli   = self.create_client(CommandBool, '/mavros/cmd/arming')
        self.land_cli     = self.create_client(Trigger, '/mavros/cmd/land')

        # ── Main control timer ───────────────────────────────────────
        # Fires every 0.1 s (10 Hz). All state machine logic lives here.
        self.create_timer(1.0 / SETPOINT_RATE_HZ, self.control_loop)

        self.get_logger().info('FlightController initialised — waiting for FCU connection...')

    # ── Callbacks ────────────────────────────────────────────────────

    def state_cb(self, msg: State):
        self.connected = msg.connected
        self.armed     = msg.armed
        self.fcu_mode  = msg.mode

    def pose_cb(self, msg: PoseStamped):
        # Local position in ENU frame (MAVROS2 converts from NED automatically)
        self.local_x = msg.pose.position.x
        self.local_y = msg.pose.position.y
        self.local_z = msg.pose.position.z

    # ── Helpers ──────────────────────────────────────────────────────

    def distance_to_wp(self, wp) -> float:
        """3D Euclidean distance from current position to a waypoint."""
        dx = wp[0] - self.local_x
        dy = wp[1] - self.local_y
        dz = wp[2] - self.local_z
        return math.sqrt(dx*dx + dy*dy + dz*dz)

    def publish_setpoint(self, x, y, z):
        """Publish a single local position setpoint."""
        sp = PoseStamped()
        sp.header.stamp    = self.get_clock().now().to_msg()
        sp.header.frame_id = 'map'
        sp.pose.position.x = float(x)
        sp.pose.position.y = float(y)
        sp.pose.position.z = float(z)
        # Orientation: identity quaternion (no yaw command)
        sp.pose.orientation.w = 1.0
        self.sp_pub.publish(sp)

    def call_set_mode(self, mode: str):
        """Send a mode change request. Returns immediately (non-blocking)."""
        req = SetMode.Request()
        req.custom_mode = mode
        future = self.set_mode_cli.call_async(req)
        # We don't wait for the future here — the next control loop tick will
        # check self.fcu_mode to confirm the change took effect.
        self.get_logger().info(f'Requesting mode: {mode}')

    def call_arming(self, arm: bool):
        """Send arm/disarm command."""
        req = CommandBool.Request()
        req.value = arm
        self.arming_cli.call_async(req)
        self.get_logger().info(f'{"Arming" if arm else "Disarming"} request sent')

    def call_land(self):
        """Trigger PX4 autonomous landing."""
        req = Trigger.Request()
        self.land_cli.call_async(req)
        self.get_logger().info('Land command sent')

    # ── State machine ────────────────────────────────────────────────

    def control_loop(self):
        """
        Called at 10 Hz. Drives the state machine.
        Each state publishes a setpoint and checks the transition condition.
        """
        if not self.connected:
            return   # FCU not connected yet — do nothing

        wp = WAYPOINTS[min(self.wp_index, len(WAYPOINTS)-1)]

        # Always publish the current target setpoint — keeps Offboard mode alive
        self.publish_setpoint(*wp)

        # ── IDLE: wait for FCU connection ─────────────────────────────
        if self.state == 'IDLE':
            self.get_logger().info('FCU connected. Starting setpoint stream...')
            self.state = 'STREAMING'

        # ── STREAMING: send ≥10 setpoints before mode switch ─────────
        elif self.state == 'STREAMING':
            self.stream_count += 1
            if self.stream_count >= 20:   # 20 × 0.1s = 2 seconds of streaming
                self.state = 'OFFBOARD'
                self.call_set_mode('OFFBOARD')

        # ── OFFBOARD: wait for mode confirmation ─────────────────────
        elif self.state == 'OFFBOARD':
            if self.fcu_mode == 'OFFBOARD':
                self.get_logger().info('OFFBOARD mode confirmed. Arming...')
                self.state = 'ARMING'
                self.call_arming(True)

        # ── ARMING: wait for armed confirmation ───────────────────────
        elif self.state == 'ARMING':
            if self.armed:
                self.get_logger().info('Armed. Taking off to WP0...')
                self.state = 'FLYING'

        # ── FLYING: navigate waypoints in sequence ────────────────────
        elif self.state == 'FLYING':
            dist = self.distance_to_wp(wp)
            self.get_logger().info(
                f'WP{self.wp_index}: dist={dist:.2f}m | pos=({self.local_x:.1f}, {self.local_y:.1f}, {self.local_z:.1f})',
                throttle_duration_sec=1.0   # print at most once per second
            )
            if dist < ACCEPTANCE_RADIUS_M:
                self.wp_index += 1
                if self.wp_index >= len(WAYPOINTS):
                    self.get_logger().info('All waypoints reached. Landing...')
                    self.state = 'LANDING'
                    self.call_land()
                else:
                    self.get_logger().info(f'WP{self.wp_index-1} reached → flying to WP{self.wp_index}')

        # ── LANDING: wait for disarm (PX4 disarms after touchdown) ────
        elif self.state == 'LANDING':
            if not self.armed:
                self.get_logger().info('Landed and disarmed. Mission complete.')
                self.state = 'DONE'

        # ── DONE: nothing more to do ──────────────────────────────────
        elif self.state == 'DONE':
            pass


def main(args=None):
    rclpy.init(args=args)
    node = FlightController()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()


if __name__ == '__main__':
    main()

Step 05: offboard_velocity_node.py

Position setpoints tell the drone where to go. Velocity setpoints tell it how fast and in which direction. This is the right interface for path following, obstacle avoidance, and smooth coordinated motion — all of which appear in later labs. This node flies a 20 m square using constant velocity commands.

STEP 05 Write offboard_velocity_node.py
Why velocity setpoints instead of position setpoints here

Position setpoints delegate path shape to PX4's internal trajectory generator — PX4 decides how fast to fly and how to curve between waypoints. Velocity setpoints give you direct control of speed and direction at every instant. This matters when you need to maintain a specific speed (formation keeping), follow a curved path (orbit), or react to real-time sensor data (obstacle avoidance). L06's vision-guided approach uses velocity setpoints to keep a detected target centred.

python · offboard_velocity_node.py 
#!/usr/bin/env python3
"""
offboard_velocity_node.py
─────────────────────────
Flies a 20 m × 20 m square at 3 m/s using velocity setpoints.
Demonstrates the difference from position setpoints — the drone
moves at a controlled speed rather than PX4-chosen trajectory.

Setpoint topic: /mavros/setpoint_velocity/cmd_vel_unstamped
Message type:   geometry_msgs/Twist
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from geometry_msgs.msg import Twist, PoseStamped
from mavros_msgs.msg   import State
from mavros_msgs.srv   import CommandBool, SetMode
from std_srvs.srv      import Trigger

import math
import time


CRUISE_SPEED_MPS  = 3.0   # side speed [m/s]
CLIMB_SPEED_MPS   = 2.0   # takeoff climb speed [m/s]
TARGET_ALTITUDE_M = 15.0  # cruise altitude [m]
SIDE_LENGTH_M     = 20.0  # square side length [m]
HOVER_DURATION_S  = 2.0   # pause at each corner [s]
RATE_HZ           = 20.0  # velocity command rate


class OffboardVelocityNode(Node):

    def __init__(self):
        super().__init__('offboard_velocity_node')

        self.state      = 'IDLE'
        self.phase      = 0     # which side of the square (0–3)
        self.phase_dist = 0.0  # distance covered in current phase
        self.hover_t    = 0.0  # time when hover started
        self.stream_cnt = 0

        self.connected  = False
        self.armed      = False
        self.fcu_mode   = ''
        self.local_z    = 0.0

        rel = QoSProfile(reliability=ReliabilityPolicy.RELIABLE, history=HistoryPolicy.KEEP_LAST, depth=10)
        sen = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT, history=HistoryPolicy.KEEP_LAST, depth=1)

        self.create_subscription(State, '/mavros/state', self.state_cb, rel)
        self.create_subscription(PoseStamped, '/mavros/local_position/pose', self.pose_cb, sen)

        # Velocity publisher — Twist message: linear.x=north, linear.y=east, linear.z=up
        self.vel_pub = self.create_publisher(Twist, '/mavros/setpoint_velocity/cmd_vel_unstamped', 10)

        self.set_mode_cli = self.create_client(SetMode, '/mavros/set_mode')
        self.arming_cli   = self.create_client(CommandBool, '/mavros/cmd/arming')
        self.land_cli     = self.create_client(Trigger, '/mavros/cmd/land')

        self.create_timer(1.0 / RATE_HZ, self.control_loop)
        self.get_logger().info('OffboardVelocityNode ready')

    def state_cb(self, msg): self.connected, self.armed, self.fcu_mode = msg.connected, msg.armed, msg.mode
    def pose_cb(self, msg): self.local_z = msg.pose.position.z

    def publish_vel(self, vx=0.0, vy=0.0, vz=0.0):
        """Publish a velocity setpoint. vx=north, vy=east, vz=up (ENU)."""
        cmd = Twist()
        cmd.linear.x = float(vx)
        cmd.linear.y = float(vy)
        cmd.linear.z = float(vz)
        self.vel_pub.publish(cmd)

    def control_loop(self):
        if not self.connected:
            return

        dt = 1.0 / RATE_HZ

        if self.state == 'IDLE':
            self.publish_vel()   # zero velocity — still streams to satisfy offboard requirement
            self.stream_cnt += 1
            if self.stream_cnt >= 20:
                self.state = 'OFFBOARD'
                req = SetMode.Request(); req.custom_mode = 'OFFBOARD'
                self.set_mode_cli.call_async(req)

        elif self.state == 'OFFBOARD':
            self.publish_vel()
            if self.fcu_mode == 'OFFBOARD':
                self.state = 'ARMING'
                req = CommandBool.Request(); req.value = True
                self.arming_cli.call_async(req)

        elif self.state == 'ARMING':
            self.publish_vel()
            if self.armed:
                self.state = 'CLIMB'
                self.get_logger().info('Armed — climbing...')

        elif self.state == 'CLIMB':
            # Climb at constant upward velocity until target altitude reached
            alt_error = TARGET_ALTITUDE_M - self.local_z
            if alt_error > 0.5:
                self.publish_vel(vz=CLIMB_SPEED_MPS)
            else:
                self.get_logger().info(f'At altitude {self.local_z:.1f}m — starting square')
                self.state     = 'SQUARE'
                self.phase     = 0
                self.phase_dist= 0.0

        elif self.state == 'SQUARE':
            # Four sides of the square, each SIDE_LENGTH_M long.
            # Velocity direction rotates 90° every phase.
            # Phase 0: north  Phase 1: east  Phase 2: south  Phase 3: west
            vx_table = [CRUISE_SPEED_MPS, 0.0, -CRUISE_SPEED_MPS, 0.0]
            vy_table = [0.0, CRUISE_SPEED_MPS, 0.0, -CRUISE_SPEED_MPS]

            if self.phase < 4:
                # Maintain altitude with a small correction term
                alt_correction = (TARGET_ALTITUDE_M - self.local_z) * 0.5
                self.publish_vel(vx=vx_table[self.phase],
                                 vy=vy_table[self.phase],
                                 vz=alt_correction)
                self.phase_dist += CRUISE_SPEED_MPS * dt

                if self.phase_dist >= SIDE_LENGTH_M:
                    self.get_logger().info(f'Side {self.phase+1}/4 done — hovering...')
                    self.phase      += 1
                    self.phase_dist  = 0.0
                    self.hover_t     = time.time()
                    self.state       = 'CORNER_HOVER'
            else:
                self.get_logger().info('Square complete — landing')
                self.state = 'LAND'

        elif self.state == 'CORNER_HOVER':
            # Hover at corner for HOVER_DURATION_S seconds, then continue square
            self.publish_vel(vz=(TARGET_ALTITUDE_M - self.local_z) * 0.5)
            if time.time() - self.hover_t >= HOVER_DURATION_S:
                self.state = 'SQUARE'

        elif self.state == 'LAND':
            self.publish_vel()
            req = Trigger.Request()
            self.land_cli.call_async(req)
            self.state = 'DONE'

        elif self.state == 'DONE':
            self.publish_vel()   # keep streaming zeros


def main(args=None):
    rclpy.init(args=args)
    node = OffboardVelocityNode()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

Step 06: mode_manager.py

The flight controller and velocity node both switch modes internally. In a real system, mode switching must be centralised with pre-flight checks — you should not enter OFFBOARD mode if GPS is unavailable, if battery is critical, or if the FCU is not healthy. The mode manager provides a service interface for safe mode transitions that the GCS (L07) and other nodes can call.

STEP 06 Write mode_manager.py
Why a dedicated mode manager?

Without a centralised mode manager, any node can switch modes at any time — the velocity node could switch to OFFBOARD while the flight controller is mid-landing, producing dangerous conflicting commands. A mode manager serialises all mode change requests, performs pre-flight checks before allowing dangerous transitions, and logs every transition with a timestamp for post-flight analysis. This is exactly how PX4's own commander module works internally — a single arbitrated state machine that all other components request transitions from.

python · mode_manager.py 
#!/usr/bin/env python3
"""
mode_manager.py
───────────────
Centralised flight mode arbitration with pre-flight checks.
Exposes a service: /request_mode (std_srvs/srv/SetBool is too simple;
we reuse Trigger with the mode encoded in a topic-based request).

For simplicity, clients publish the requested mode string to
/mode_manager/request (std_msgs/String), and mode_manager evaluates
whether the transition is safe before calling MAVROS2.

Pre-flight checks performed before OFFBOARD:
  1. FCU connected
  2. GPS fix available
  3. Battery above BATTERY_MIN_PCT
  4. No recent IMU stale flag
"""

import rclpy
from rclpy.node          import Node
from rclpy.executors     import MultiThreadedExecutor
from rclpy.callback_group import ReentrantCallbackGroup
from rclpy.qos           import QoSProfile, ReliabilityPolicy, HistoryPolicy

from std_msgs.msg   import String
from std_srvs.srv   import Trigger
from sensor_msgs.msg import NavSatFix, BatteryState
from mavros_msgs.msg import State
from mavros_msgs.srv import SetMode, CommandBool

import time

BATTERY_MIN_PCT = 20.0   # minimum battery % to allow OFFBOARD entry


class ModeManager(Node):

    def __init__(self):
        super().__init__('mode_manager')
        cg = ReentrantCallbackGroup()

        rel = QoSProfile(reliability=ReliabilityPolicy.RELIABLE, history=HistoryPolicy.KEEP_LAST, depth=10)
        sen = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT, history=HistoryPolicy.KEEP_LAST, depth=1)

        # ── Monitoring state ─────────────────────────────────────────
        self.fcu_connected  = False
        self.fcu_mode       = 'UNKNOWN'
        self.armed          = False
        self.gps_fix        = -1
        self.battery_pct    = 100.0
        self.last_imu_t     = time.time()
        self.mode_history   = []   # list of (timestamp, from_mode, to_mode, result)

        # ── Subscriptions ────────────────────────────────────────────
        self.create_subscription(State, '/mavros/state',
            self.state_cb, rel, callback_group=cg)
        self.create_subscription(NavSatFix, '/mavros/global_position/global',
            self.gps_cb, sen, callback_group=cg)
        self.create_subscription(BatteryState, '/mavros/battery',
            self.battery_cb, sen, callback_group=cg)

        # Listen for mode requests on a topic (String: 'OFFBOARD', 'AUTO.LAND', etc.)
        self.create_subscription(String, '/mode_manager/request',
            self.mode_request_cb, rel, callback_group=cg)

        # ── Services: expose pre-flight check and mode history ────────
        self.create_service(Trigger, '/mode_manager/preflight_check',
            self.preflight_check_srv, callback_group=cg)
        self.create_service(Trigger, '/mode_manager/get_history',
            self.history_srv, callback_group=cg)

        # ── MAVROS2 service clients ───────────────────────────────────
        self.set_mode_cli = self.create_client(SetMode, '/mavros/set_mode')
        self.arming_cli   = self.create_client(CommandBool, '/mavros/cmd/arming')

        self.get_logger().info('ModeManager ready. Listening on /mode_manager/request')

    def state_cb(self, msg: State):
        self.fcu_connected = msg.connected
        self.armed         = msg.armed
        self.fcu_mode      = msg.mode

    def gps_cb(self, msg: NavSatFix):
        self.gps_fix = msg.status.status

    def battery_cb(self, msg: BatteryState):
        self.battery_pct = msg.percentage * 100.0

    def _preflight_ok(self) -> tuple:
        """Returns (ok: bool, reason: str)."""
        if not self.fcu_connected:
            return False, 'FCU not connected'
        if self.gps_fix < 0:
            return False, 'No GPS fix'
        if self.battery_pct < BATTERY_MIN_PCT:
            return False, f'Battery too low: {self.battery_pct:.1f}%'
        if time.time() - self.last_imu_t > 2.0:
            return False, 'IMU data stale'
        return True, 'All checks passed'

    def mode_request_cb(self, msg: String):
        """
        Receives mode change requests. Performs pre-flight checks for
        safety-critical modes before calling MAVROS2 set_mode.
        """
        requested = msg.data.strip().upper()
        from_mode = self.fcu_mode
        t_now     = time.strftime('%H:%M:%S')

        # Modes that require pre-flight clearance
        NEEDS_CHECK = {'OFFBOARD', 'AUTO.MISSION', 'AUTO.TAKEOFF'}

        if requested in NEEDS_CHECK:
            ok, reason = self._preflight_ok()
            if not ok:
                self.get_logger().warn(
                    f'[{t_now}] Mode {requested} BLOCKED: {reason}')
                self.mode_history.append((t_now, from_mode, requested, f'BLOCKED: {reason}'))
                return

        # Execute the mode change
        req = SetMode.Request()
        req.custom_mode = requested
        self.set_mode_cli.call_async(req)
        self.get_logger().info(f'[{t_now}] Mode change: {from_mode} → {requested}')
        self.mode_history.append((t_now, from_mode, requested, 'REQUESTED'))

    def preflight_check_srv(self, request, response):
        ok, reason = self._preflight_ok()
        response.success = ok
        response.message = reason
        return response

    def history_srv(self, request, response):
        lines = [f'{t} | {fr} → {to} | {res}'
                 for t, fr, to, res in self.mode_history[-20:]]
        response.success = True
        response.message = '\n'.join(lines) if lines else 'No mode changes yet'
        return response


def main(args=None):
    rclpy.init(args=args)
    node = ModeManager()
    executor = MultiThreadedExecutor(num_threads=4)
    executor.add_node(node)
    try: executor.spin()
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

Step 07: emergency_stop.py

The most important node in the system. One topic message triggers an immediate, unconditional land — bypassing all other logic, all other nodes, no questions asked. Every serious autonomous UAV system must have this, and it must be the simplest, most reliable component in the stack.

STEP 07 Write emergency_stop.py
Design principle — emergency stops must be simple

An emergency stop node should have zero dependencies on the state of other nodes. It should not wait for the health service, not check the mode manager, not look up the GPS fix. It receives one message and does one thing: call the land service. Complexity is the enemy of reliability in safety-critical code. This node is deliberately minimal — 40 lines, one subscription, one service call, no state.

#!/usr/bin/env python3
"""
emergency_stop.py
─────────────────
Subscribes to /emergency_stop (std_msgs/Bool).
On any True message: immediately triggers AUTO.LAND via MAVROS2.

This node is intentionally minimal — no pre-flight checks,
no state, no dependencies. Simple = reliable.

Trigger from terminal:
  ros2 topic pub --once /emergency_stop std_msgs/msg/Bool "data: true"
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from std_msgs.msg  import Bool
from std_srvs.srv  import Trigger
from mavros_msgs.srv import CommandBool


class EmergencyStop(Node):

    def __init__(self):
        super().__init__('emergency_stop')

        # Use RELIABLE QoS — we cannot afford to miss the stop command
        rel = QoSProfile(
            reliability=ReliabilityPolicy.RELIABLE,
            history=HistoryPolicy.KEEP_LAST,
            depth=10
        )

        self.create_subscription(Bool, '/emergency_stop',
                                 self.estop_cb, rel)

        self.land_cli    = self.create_client(Trigger, '/mavros/cmd/land')
        self.disarm_cli  = self.create_client(CommandBool, '/mavros/cmd/arming')

        self._triggered = False   # prevent repeated calls

        self.get_logger().info(
            'EmergencyStop ACTIVE — listening on /emergency_stop')

    def estop_cb(self, msg: Bool):
        if not msg.data or self._triggered:
            return

        self._triggered = True
        self.get_logger().error(
            '!!! EMERGENCY STOP TRIGGERED !!! Commanding immediate land.')

        # Step 1: command land (gentle, controlled descent)
        land_req = Trigger.Request()
        self.land_cli.call_async(land_req)

        # Step 2: schedule disarm 5 s later as a fallback
        # (if land service fails, disarm cuts motors directly)
        self.create_timer(5.0, self._fallback_disarm')

    def _fallback_disarm(self):
        self.get_logger().warn('Fallback: sending disarm command')
        req = CommandBool.Request()
        req.value = False   # disarm
        self.disarm_cli.call_async(req)


def main(args=None):
    rclpy.init(args=args)
    node = EmergencyStop()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

#!/usr/bin/env python3
"""
mission_runner.py
─────────────────
Integration demo: orchestrates pre-flight check → flight controller
→ L02 action server → land, with emergency stop active throughout.

Run with:
  ros2 launch aeroknow_bringup l03_launch.py
  # (which starts mission_runner + mode_manager + emergency_stop + flight_controller)
"""

import rclpy
from rclpy.node   import Node
from rclpy.action import ActionClient

from std_msgs.msg  import String
from std_srvs.srv  import Trigger
from aeroknow_bringup.action import FlyToWaypoint

import time


DEMO_WAYPOINTS = [
    (47.3978, 8.5456, 25.0),   # WP1 — north of SITL default origin
    (47.3982, 8.5464, 25.0),   # WP2 — northeast
    (47.3975, 8.5470, 20.0),   # WP3 — east, descend slightly
]


class MissionRunner(Node):

    def __init__(self):
        super().__init__('mission_runner')

        # Service clients
        self.preflight_cli = self.create_client(Trigger, '/mode_manager/preflight_check')
        self.mode_req_pub  = self.create_publisher(String, '/mode_manager/request', 10)

        # Action client for L02 FlyToWaypoint
        self._wp_client = ActionClient(self, FlyToWaypoint, 'fly_to_waypoint')

        self.get_logger().info('MissionRunner starting in 3s...')
        self.create_timer(3.0, self._run_once')   # delay start to allow other nodes to start
        self._started = False

    def _run_once(self):
        if self._started: return
        self._started = True

        # Step 1 — Pre-flight check
        self.get_logger().info('Running pre-flight check...')
        if not self.preflight_cli.wait_for_service(timeout_sec=5.0):
            self.get_logger().error('Pre-flight service unavailable')
            return

        future = self.preflight_cli.call_async(Trigger.Request())
        rclpy.spin_until_future_complete(self, future, timeout_sec=5.0)
        result = future.result()

        if result is None or not result.success:
            reason = result.message if result else 'timeout'
            self.get_logger().error(f'Pre-flight FAILED: {reason} — mission aborted')
            return

        self.get_logger().info(f'Pre-flight OK: {result.message}')

        # Step 2 — Request OFFBOARD mode via mode manager
        mode_msg = String(); mode_msg.data = 'OFFBOARD'
        self.mode_req_pub.publish(mode_msg)
        self.get_logger().info('OFFBOARD mode requested via mode manager')
        time.sleep(1.0)

        # Step 3 — Fly each waypoint using the L02 action server
        if not self._wp_client.wait_for_server(timeout_sec=5.0):
            self.get_logger().error('FlyToWaypoint action server not available')
            return

        for i, (lat, lon, alt) in enumerate(DEMO_WAYPOINTS):
            self.get_logger().info(f'Sending WP{i+1}: ({lat:.4f}, {lon:.4f}, {alt:.0f}m)')

            goal = FlyToWaypoint.Goal()
            goal.target_lat       = lat
            goal.target_lon       = lon
            goal.target_alt       = alt
            goal.acceptance_radius= 5.0

            future = self._wp_client.send_goal_async(
                goal,
                feedback_callback=lambda fb: self.get_logger().info(
                    f'  WP{i+1} feedback: {fb.feedback.distance_to_wp:.1f}m remaining',
                    throttle_duration_sec=3.0
                )
            )
            rclpy.spin_until_future_complete(self, future, timeout_sec=5.0)
            goal_handle = future.result()

            if not goal_handle.accepted:
                self.get_logger().error(f'WP{i+1} rejected')
                continue

            result_future = goal_handle.get_result_async()
            rclpy.spin_until_future_complete(self, result_future, timeout_sec=120.0)
            res = result_future.result().result

            if res.success:
                self.get_logger().info(f'WP{i+1} reached (dist={res.final_distance:.1f}m)')
            else:
                self.get_logger().warn(f'WP{i+1} result: {res.message}')

        self.get_logger().info('Mission complete.')


def main(args=None):
    rclpy.init(args=args)
    node = MissionRunner()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

Common Errors & Fixes

Error 01 — Mode never switches to OFFBOARD Requesting mode: OFFBOARD ← printed, but fcu_mode stays 'POSCTL'

Cause: Setpoints were not streaming before the mode switch request. PX4 requires at least ~0.5 s of continuous setpoint messages at ≥2 Hz before accepting OFFBOARD. Fix: Confirm the STREAMING state in flight_controller.py waits for stream_count >= 20 (2 seconds at 10 Hz) before calling call_set_mode. If you modified the code and shortened this wait, put it back. You can verify setpoints are flowing: ros2 topic hz /mavros/setpoint_position/local should show ~10 Hz before mode switch.

Error 02 — Drone won't arm: "Preflight fail" commander: Preflight Fail: Gyro Sensor 0 uncalibrated

Cause: SITL occasionally fails IMU calibration on startup. Fix: In the PX4 shell (Terminal 1), run: commander calibrate gyro then wait 5 s. If it still fails, restart PX4 SITL entirely: Ctrl+C in Terminal 1, then make px4_sitl gz_x500 again. This is a SITL quirk — it does not occur on real hardware with pre-calibrated sensors.

Error 03 — Drone arms but won't takeoff (stays on ground) [INFO] Armed. Taking off to WP0... (no movement in Gazebo)

Cause: The setpoint Z value is in ENU (+Z = up), but a value of 0.0 means "hover at ground level" — the drone hovers right where it is. Fix: Confirm WAYPOINTS[0] has z = 20.0 (or similar positive value), not 0.0. Also check that publish_setpoint is still being called inside the ARMING state — if you accidentally removed the setpoint publish before the state check, Offboard mode times out and PX4 exits back to POSCTL before arming completes.

Error 04 — mission_runner can't find action server Waiting for FlyToWaypoint action server... (hangs)

Cause: mission_action_server is not running, or it started but failed due to the aeroknow_bringup.action import error (L02 Error 02). Fix: In a separate terminal: ros2 action list — if /fly_to_waypoint is not listed, the action server crashed. Check its terminal for errors. Most likely cause: colcon build was not run after adding the action file, or source install/setup.bash was not run in the terminal where the launch file was invoked.

Error 05 — Emergency stop triggered but drone keeps flying [ERROR] !!! EMERGENCY STOP TRIGGERED !!! ... (drone ignores it)

Cause: The /mavros/cmd/land service client was not yet connected when the stop was triggered. This happens if emergency_stop.py receives the stop signal within the first few hundred milliseconds of startup before MAVROS2 has registered the service. Fix: Add a self.land_cli.wait_for_service(timeout_sec=3.0) call inside __init__ — wait for MAVROS2 services to become available before the node reports ready. The node startup log should show the service is available before ACTIVE is printed.

Extension Challenges

The full flight stack is running. Here are three harder things to try before L04.

  1. Add a loiter function to flight_controller.py. After reaching each waypoint, the drone should orbit in a 10 m radius circle for 15 seconds before moving to the next. Use velocity setpoints to hold altitude and command tangential velocity (vx = speed × cos(θ), vy = speed × sin(θ) where θ advances at a constant rate). This is the foundational building block for ISR orbit patterns used in L06.
  2. Extend mode_manager.py with a transition guard table. Not all mode transitions should be allowed at all times — for example, switching from AUTO.LAND back to OFFBOARD mid-descent is dangerous. Define a Python dict ALLOWED_TRANSITIONS that maps (from_mode, to_mode) → bool, and check it in mode_request_cb before executing. Log blocked transitions with a clear reason.
  3. Add a Return-to-Base (RTB) command to emergency_stop.py. Currently the emergency stop commands immediate land — which could be dangerous over water or occupied terrain. Add a second topic /rtb_command (std_msgs/Bool) that instead commands AUTO.RTL (return to launch). RTL uses the home position set at arm time. Test by publishing to /rtb_command while the mission is running in Gazebo — the drone should return to its takeoff position and land there.
