Architecture — The Two-Layer Design

The GCS has a strict two-layer design. This is not optional — it is necessary because PyQt5's UI thread must never block, and ROS2's spin() must never be called from the UI thread. The layers communicate through Qt signals and slots, which are thread-safe by design.

The Golden Rule of Qt + ROS2 All ROS2 work happens in a background thread. All UI updates happen in the Qt main thread. The bridge between them is Qt signals. The backend node runs rclpy.spin() in a threading.Thread. When a ROS2 callback receives data, it emits a Qt signal. Qt routes that signal to a slot in the UI thread, which updates the widget. Breaking this rule — calling a Qt widget method from a ROS2 callback thread — causes random crashes that are nearly impossible to debug.

Step 02: gcs_backend.py — The ROS2 Bridge Layer

The backend is a QObject subclass that runs a ROS2 node in a background thread and exposes the data to the UI through Qt signals. Every ROS2 subscription emits a signal. Every UI command call goes through a method on this object. The UI never touches rclpy directly.

STEP 02 Write gcs_backend.py
Why QObject and not just a plain Python class

Qt signals and slots only work between QObject subclasses. A signal emitted from a background thread is automatically queued and delivered to the UI thread — this is thread-safe communication built into Qt's event loop. If you used a plain Python queue or callback instead, you would need to poll it from a UI timer, introducing latency and complexity. The QObject + signal approach is both simpler and safer.

#!/usr/bin/env python3
"""
gcs_backend.py
──────────────
ROS2 node wrapped as a QObject.
Runs rclpy.spin() in a background thread.
Exposes all drone data as Qt signals.
Receives UI commands and forwards them to ROS2.

Threading model:
  Main thread:    Qt event loop (UI)
  Backend thread: rclpy.spin(node) — all ROS2 callbacks fire here
  Bridge:         Qt signals (thread-safe cross-thread delivery)
"""

import rclpy
from rclpy.node           import Node
from rclpy.action         import ActionClient
from rclpy.qos            import QoSProfile, ReliabilityPolicy, HistoryPolicy
from rclpy.executors      import MultiThreadedExecutor

from PyQt5.QtCore         import QObject, pyqtSignal, QThread

from mavros_msgs.msg      import State
from mavros_msgs.srv      import CommandBool, SetMode
from sensor_msgs.msg      import Imu, NavSatFix, BatteryState, Image
from geometry_msgs.msg    import PoseStamped
from std_msgs.msg         import Bool, String, Float32
from vision_msgs.msg      import Detection2DArray
from geographic_msgs.msg  import GeoPoint
from std_srvs.srv         import Trigger

from aeroknow_bringup.action import FlyToWaypoint

import threading
import math


class GCSBackend(QObject):
    """
    Signals emitted to the UI thread:
      All carry Python primitives or dicts — never ROS2 message objects.
      This keeps the UI layer decoupled from rclpy.
    """
    # Telemetry
    sig_state      = pyqtSignal(dict)    # {connected, armed, mode}
    sig_position   = pyqtSignal(dict)    # {lat, lon, alt_rel, alt_abs}
    sig_attitude   = pyqtSignal(dict)    # {roll, pitch, yaw}
    sig_battery    = pyqtSignal(dict)    # {voltage, percentage}
    sig_velocity   = pyqtSignal(dict)    # {vx, vy, vz, groundspeed}
    # Navigation health (L05)
    sig_nav_health = pyqtSignal(str)     # raw string from /nav_health
    sig_gps_denied = pyqtSignal(bool)
    # Vision (L06)
    sig_detections = pyqtSignal(list)    # list of dicts
    sig_video_frame= pyqtSignal(object)  # numpy BGR array
    sig_target_gps = pyqtSignal(float, float)  # lat, lon
    # Feedback
    sig_log        = pyqtSignal(str)     # timestamped log message
    sig_fps        = pyqtSignal(float)

    def __init__(self, parent=None):
        super().__init__(parent)
        rclpy.init()
        self._node     = _GCSNode(self)
        self._executor = MultiThreadedExecutor(num_threads=4)
        self._executor.add_node(self._node)
        self._thread   = threading.Thread(target=self._spin, daemon=True)
        self._thread.start()

    def _spin(self):
        try:
            self._executor.spin()
        except Exception as e:
            self.sig_log.emit(f'[ERROR] ROS2 spin error: {e}')

    def shutdown(self):
        self._executor.shutdown()
        rclpy.shutdown()

    # ── Commands (called from UI thread, forwarded to ROS2) ──────────

    def arm(self):
        self._node.send_arm(True)
        self.sig_log.emit('CMD: Arm requested')

    def disarm(self):
        self._node.send_arm(False)
        self.sig_log.emit('CMD: Disarm requested')

    def set_mode(self, mode: str):
        self._node.send_mode(mode)
        self.sig_log.emit(f'CMD: Mode → {mode}')

    def emergency_stop(self):
        self._node.send_estop()
        self.sig_log.emit('CMD: *** EMERGENCY STOP ***')

    def fly_to_waypoint(self, lat: float, lon: float, alt: float):
        self._node.send_waypoint(lat, lon, alt)
        self.sig_log.emit(f'CMD: Waypoint → ({lat:.6f}, {lon:.6f}, {alt:.1f}m)')


class _GCSNode(Node):
    """
    The actual rclpy Node. Holds all subscriptions and service clients.
    Calls backend.sig_* to pass data to the UI.
    """

    def __init__(self, backend: GCSBackend):
        super().__init__('aeroknow_gcs_node')
        self.be = backend

        sen = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                         history=HistoryPolicy.KEEP_LAST, depth=1)
        rel = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                         history=HistoryPolicy.KEEP_LAST, depth=10)

        # ── Subscriptions ────────────────────────────────────────────
        self.create_subscription(State,    '/mavros/state',              self._state_cb,    rel)
        self.create_subscription(NavSatFix,'/mavros/global_position/global',self._gps_cb,      sen)
        self.create_subscription(PoseStamped,'/mavros/local_position/pose', self._pose_cb,     sen)
        self.create_subscription(BatteryState,'/mavros/battery',           self._battery_cb,  sen)
        self.create_subscription(Imu,      '/mavros/imu/data',            self._imu_cb,      sen)
        self.create_subscription(String,   '/nav_health',                 self._health_cb,   rel)
        self.create_subscription(Bool,     '/nav/gps_denied',             self._denied_cb,   rel)
        self.create_subscription(Detection2DArray,'/vision/detections',   self._dets_cb,     rel)
        self.create_subscription(Image,    '/vision/overlay',             self._video_cb,    sen)
        self.create_subscription(GeoPoint, '/targets/gps',               self._target_cb,   rel)
        self.create_subscription(Float32,  '/vision/fps',                 self._fps_cb,      rel)

        # ── Service clients ──────────────────────────────────────────
        self._arm_cli  = self.create_client(CommandBool, '/mavros/cmd/arming')
        self._mode_cli = self.create_client(SetMode,     '/mavros/set_mode')
        self._land_cli = self.create_client(Trigger,     '/mavros/cmd/land')

        # ── Publishers ───────────────────────────────────────────────
        self._estop_pub = self.create_publisher(Bool, '/emergency_stop', 10)
        self._mode_pub  = self.create_publisher(String, '/mode_manager/request', 10)

        # ── Action client ────────────────────────────────────────────
        self._wp_client = ActionClient(self, FlyToWaypoint, 'fly_to_waypoint')

        # ── cv_bridge (for video) ────────────────────────────────────
        try:
            from cv_bridge import CvBridge
            self._bridge = CvBridge()
        except ImportError:
            self._bridge = None

        self._alt_rel = 0.0

    # ── Subscription callbacks ───────────────────────────────────────

    def _state_cb(self, m: State):
        self.be.sig_state.emit({
            'connected': m.connected, 'armed': m.armed, 'mode': m.mode})

    def _gps_cb(self, m: NavSatFix):
        self.be.sig_position.emit({
            'lat': m.latitude, 'lon': m.longitude,
            'alt_abs': m.altitude, 'alt_rel': self._alt_rel})

    def _pose_cb(self, m: PoseStamped):
        self._alt_rel = m.pose.position.z
        # Extract roll, pitch, yaw from quaternion
        q = m.pose.orientation
        sinr = 2*(q.w*q.x + q.y*q.z); cosr = 1 - 2*(q.x**2 + q.y**2)
        roll  = math.degrees(math.atan2(sinr, cosr))
        sinp  = 2*(q.w*q.y - q.z*q.x)
        pitch = math.degrees(math.asin(max(-1, min(1, sinp))))
        siny  = 2*(q.w*q.z + q.x*q.y); cosy = 1 - 2*(q.y**2 + q.z**2)
        yaw   = math.degrees(math.atan2(siny, cosy))
        self.be.sig_attitude.emit({'roll': roll, 'pitch': pitch, 'yaw': yaw})

    def _battery_cb(self, m: BatteryState):
        self.be.sig_battery.emit({
            'voltage': m.voltage, 'percentage': m.percentage * 100})

    def _imu_cb(self, m: Imu):
        # IMU linear acceleration ≈ body velocity changes; we just forward for display
        ax, ay = m.linear_acceleration.x, m.linear_acceleration.y
        gs = math.sqrt(ax**2 + ay**2)
        self.be.sig_velocity.emit({'vx': ax, 'vy': ay, 'vz': m.linear_acceleration.z, 'groundspeed': gs})

    def _health_cb(self, m: String): self.be.sig_nav_health.emit(m.data)
    def _denied_cb(self, m: Bool):   self.be.sig_gps_denied.emit(m.data)
    def _target_cb(self, m: GeoPoint): self.be.sig_target_gps.emit(m.latitude, m.longitude)
    def _fps_cb(self, m: Float32):   self.be.sig_fps.emit(m.data)

    def _dets_cb(self, m: Detection2DArray):
        dets = []
        for d in m.detections:
            conf = d.results[0].hypothesis.score    if d.results else 0
            cls  = d.results[0].hypothesis.class_id if d.results else '?'
            dets.append({'class_id': cls, 'conf': conf,
                          'cx': d.bbox.center.position.x,
                          'cy': d.bbox.center.position.y,
                          'w': d.bbox.size_x, 'h': d.bbox.size_y})
        self.be.sig_detections.emit(dets)

    def _video_cb(self, m: Image):
        if not self._bridge: return
        try:
            frame = self._bridge.imgmsg_to_cv2(m, desired_encoding='bgr8')
            self.be.sig_video_frame.emit(frame)
        except Exception: pass

    # ── Command methods (called from UI, executed in ROS2 thread) ────

    def send_arm(self, arm: bool):
        req = CommandBool.Request(); req.value = arm
        self._arm_cli.call_async(req)

    def send_mode(self, mode: str):
        msg = String(); msg.data = mode
        self._mode_pub.publish(msg)

    def send_estop(self):
        msg = Bool(); msg.data = True
        self._estop_pub.publish(msg)

    def send_waypoint(self, lat: float, lon: float, alt: float):
        if not self._wp_client.wait_for_server(timeout_sec=1.0):
            self.be.sig_log.emit('[WARN] FlyToWaypoint action server not available')
            return
        goal = FlyToWaypoint.Goal()
        goal.target_lat = lat; goal.target_lon = lon
        goal.target_alt = alt; goal.acceptance_radius = 5.0
        self._wp_client.send_goal_async(goal)

Why the emergency stop button is always visible and always red

The emergency stop must never be more than one click away, regardless of which tab or panel the operator is looking at. It lives in a fixed toolbar that spans the full window width and is never hidden. The red background is a deliberate violation of normal UI aesthetics — its purpose is to be unmissable. This is a direct implementation of the operator authority principle from Module 01 §1.5: the human retains unconditional override authority at all times.

def ts() -> str:
    """Current timestamp for log entries."""
    return QDateTime.currentDateTime().toString('hh:mm:ss')


class TelemetryPanel(QGroupBox):
    """6-field live telemetry display."""

    def __init__(self):
        super().__init__('Telemetry')
        grid = QGridLayout(self)
        grid.setSpacing(8)
        self._fields = {}
        pairs = [
            ('MODE',  '—'), ('ARMED',     '—'),
            ('ALT',   '—'), ('BATTERY',   '—'),
            ('LAT',   '—'), ('LON',       '—'),
            ('ROLL',  '—'), ('PITCH',     '—'),
            ('YAW',   '—'), ('GPS DENIED', '—'),
        ]
        for i, (k, v) in enumerate(pairs):
            row, col = divmod(i, 2)
            lbl = QLabel(k + ':')
            lbl.setStyleSheet('color:#4a7a8a; font-size:11px;')
            val = QLabel(v)
            val.setObjectName('val')
            grid.addWidget(lbl, row, col*2)
            grid.addWidget(val, row, col*2+1)
            self._fields[k] = val

    def set(self, key: str, value: str, colour: str = '#00ff9d'):
        if key in self._fields:
            self._fields[key].setText(value)
            self._fields[key].setStyleSheet(f'color:{colour}; font-size:15px; font-weight:bold;')

            self.be.fly_to_waypoint(lat, lon, alt)
        except ValueError:
            QMessageBox.warning(self, 'Input Error', 'Lat/Lon/Alt must be numbers.')

Step 04: map_widget.py — Live Folium Map

Folium generates Leaflet.js HTML maps. We embed one in a QWebEngineView, then update the drone's position marker by regenerating the HTML and reloading the view. It is not the most elegant approach — but it requires zero backend infrastructure and works offline with cached tiles.

STEP 04 Write map_widget.py
Why regenerate HTML instead of updating the marker in JavaScript

The alternative — calling view.page().runJavaScript() to move a marker — requires writing a JavaScript interface between Python and the Leaflet map, and synchronising the two-way communication. For a lab application, regenerating a small HTML file every second is far simpler and perfectly fast enough. At 1 Hz position updates, the HTML is ~5 KB and the reload is imperceptible. For a production GCS, the JS interface is worth building — see the extension challenge.

#!/usr/bin/env python3
"""
map_widget.py
─────────────
Folium map embedded in a QWebEngineView.
Updates drone position + track history + target markers.
Refreshes at 1 Hz by regenerating the HTML file.
"""

import os
import tempfile
import folium

from PyQt5.QtWidgets import QWidget, QVBoxLayout, QLabel
from PyQt5.QtCore    import QTimer, pyqtSlot, QUrl

try:
    from PyQt5.QtWebEngineWidgets import QWebEngineView
    WEBENGINE_OK = True
except ImportError:
    WEBENGINE_OK = False

# Default map centre — Zurich (PX4 SITL default GPS origin)
DEFAULT_LAT = 47.3978
DEFAULT_LON = 8.5456
MAX_TRACK   = 200   # max position history points


class MapWidget(QWidget):
    """Live map widget. Falls back to a label if QtWebEngine not installed."""

    def __init__(self, backend, parent=None):
        super().__init__(parent)
        self.be      = backend
        self.lat     = DEFAULT_LAT
        self.lon     = DEFAULT_LON
        self.track   = []   # list of (lat, lon)
        self.targets = []   # list of (lat, lon)
        self._map_file = os.path.join(tempfile.gettempdir(), 'aeroknow_map.html')

        layout = QVBoxLayout(self)
        layout.setContentsMargins(0, 0, 0, 0)

        if WEBENGINE_OK:
            self._view = QWebEngineView()
            layout.addWidget(self._view)
            # Render initial map
            self._render_map()
            self._view.load(QUrl.fromLocalFile(self._map_file))

            # Refresh at 1 Hz when position changes
            self._dirty = False
            self._timer = QTimer(self)
            self._timer.timeout.connect(self._refresh)
            self._timer.start(1000)
        else:
            lbl = QLabel(
                'Map requires PyQtWebEngine.\n'
                'Install: pip3 install PyQtWebEngine\n'
                '         sudo apt install python3-pyqt5.qtwebengine'
            )
            lbl.setStyleSheet('color:#ffb700; padding:20px;')
            lbl.setAlignment(Qt.AlignCenter if True else 4)
            layout.addWidget(lbl)

        # Connect backend signals
        self.be.sig_position.connect(self._on_position)
        self.be.sig_target_gps.connect(self._on_target)

    @pyqtSlot(dict)
    def _on_position(self, d):
        self.lat = d['lat']; self.lon = d['lon']
        self.track.append((self.lat, self.lon))
        if len(self.track) > MAX_TRACK:
            self.track.pop(0)
        self._dirty = True

    @pyqtSlot(float, float)
    def _on_target(self, lat, lon):
        self.targets.append((lat, lon))
        if len(self.targets) > 20:
            self.targets.pop(0)
        self._dirty = True

    def _refresh(self):
        if not (WEBENGINE_OK and self._dirty): return
        self._dirty = False
        self._render_map()
        self._view.load(QUrl.fromLocalFile(self._map_file))

    def _render_map(self):
        """Build Folium HTML — drone position + track + targets."""
        m = folium.Map(
            location=[self.lat, self.lon],
            zoom_start=17,
            tiles='OpenStreetMap',
        )
        # Drone position marker
        folium.Marker(
            location=[self.lat, self.lon],
            tooltip=f'Drone ({self.lat:.6f}, {self.lon:.6f})',
            icon=folium.Icon(color='green', icon='plane', prefix='fa'),
        ).add_to(m)

        # Track polyline
        if len(self.track) >= 2:
            folium.PolyLine(self.track, color='#00ff9d', weight=2,
                            opacity=0.8, tooltip='Flight track').add_to(m)

        # Target markers (red)
        for i, (tlat, tlon) in enumerate(self.targets):
            folium.Marker(
                location=[tlat, tlon],
                tooltip=f'Target {i+1}',
                icon=folium.Icon(color='red', icon='crosshairs', prefix='fa'),
            ).add_to(m)

        m.save(self._map_file)

#!/usr/bin/env python3
"""
video_widget.py
───────────────
Displays OpenCV frames from /vision/overlay in a Qt widget.
Conversion chain: numpy BGR → QImage → QPixmap → scaled QLabel.
"""

import numpy as np
from PyQt5.QtWidgets import QWidget, QVBoxLayout, QLabel
from PyQt5.QtCore    import pyqtSlot, Qt
from PyQt5.QtGui     import QImage, QPixmap


class VideoWidget(QWidget):
    """
    Subscribes to sig_video_frame (numpy BGR array).
    Renders the frame scaled to fill the widget.
    """

    def __init__(self, backend, parent=None):
        super().__init__(parent)
        self.be = backend

        layout = QVBoxLayout(self)
        layout.setContentsMargins(0, 0, 0, 0)

        self._lbl = QLabel('Waiting for camera feed...')
        self._lbl.setAlignment(Qt.AlignCenter)
        self._lbl.setStyleSheet('background:#040c14; color:#3d6b7a;')
        self._lbl.setMinimumSize(320, 240)
        layout.addWidget(self._lbl)

        # FPS label overlay
        self._fps_lbl = QLabel('')
        self._fps_lbl.setStyleSheet('color:#00ff9d; font-size:11px; padding:2px;')
        layout.addWidget(self._fps_lbl)

        # Connect signals
        self.be.sig_video_frame.connect(self._on_frame)
        self.be.sig_fps.connect(self._on_fps)

    @pyqtSlot(object)
    def _on_frame(self, frame: np.ndarray):
        """
        Convert numpy BGR frame to QPixmap and display.
        Steps:
        1. Convert BGR → RGB (Qt uses RGB)
        2. Create QImage from raw bytes (must specify bytes_per_line!)
        3. Convert to QPixmap
        4. Scale to widget size maintaining aspect ratio
        """
        rgb = frame[:, :, ::-1].copy()   # BGR → RGB, must copy (contiguous array)
        h, w, ch = rgb.shape
        bytes_per_line = ch * w

        q_img = QImage(rgb.data, w, h, bytes_per_line, QImage.Format_RGB888)
        pixmap = QPixmap.fromImage(q_img)

        # Scale to widget size, maintaining aspect ratio
        scaled = pixmap.scaled(
            self._lbl.width(), self._lbl.height(),
            Qt.KeepAspectRatio, Qt.SmoothTransformation
        )
        self._lbl.setPixmap(scaled)

    @pyqtSlot(float)
    def _on_fps(self, fps: float):
        self._fps_lbl.setText(f'Inference: {fps:.1f} Hz')

Common Errors & Fixes

Error 01 — GCS crashes immediately with "QObject: Cannot create children for a parent that is in a different thread" QObject: Cannot create children for a parent that is in a different thread.

Cause: A ROS2 callback is directly calling a Qt widget method — violating the golden rule. Common location: the _video_cb calling self.lbl.setPixmap() directly instead of emitting a signal. Fix: Every UI update must go through a Qt signal. In _GCSNode, the video callback must emit self.be.sig_video_frame.emit(frame) and the widget's _on_frame slot (decorated with @pyqtSlot) does the actual QPixmap update. Never call any Qt widget method from a ROS2 callback.

Error 02 — Map tab shows "Map requires PyQtWebEngine" Map requires PyQtWebEngine. Install: pip3 install PyQtWebEngine

Cause: PyQt5.QtWebEngineWidgets is not installed. Fix: pip3 install PyQtWebEngine and sudo apt install python3-pyqt5.qtwebengine. Note: on some Ubuntu 22.04 installations, the pip package conflicts with the apt package. Install the apt version first, then check if the pip version is still needed. Confirm: python3 -c "from PyQt5.QtWebEngineWidgets import QWebEngineView; print('OK')".

Error 03 — "rclpy already initialised" when launching GCS RuntimeError: rclpy.init() called while ROS is already initialized

Cause: Two GCS instances are running, or a previous run did not shut down cleanly and left a stale ROS2 context. Fix: (1) Kill all running GCS instances: pkill -f "aeroknow_gcs". (2) Check for leftover rclpy processes: ros2 node list | grep gcs. (3) If the problem persists, add a guard in GCSBackend.__init__: if not rclpy.ok(): rclpy.init() instead of unconditional rclpy.init().

Error 04 — Arm button triggers but drone does not arm CMD: Arm requested (in log) — but ARMED field stays DISARMED

Cause: Either the drone is not in a mode that allows arming (must be in MANUAL or STABILIZED for the first arm), or PX4 has a pre-arm failure (GPS, IMU calibration). Fix: (1) Check the PX4 shell (Terminal 1) for commander: Preflight Fail messages. (2) The GCS arm button calls the raw MAVROS2 arming service — it bypasses the mode manager's pre-flight check. Add a health check call before arming in _arm(): call self.be.set_mode('OFFBOARD') first, wait for OFFBOARD confirmation via the state signal, then arm. This is exactly how QGroundControl does it.

Error 05 — Map does not update after drone takes off Green plane marker stays at origin — never moves

Cause: The sig_position signal is not connected to the map widget's _on_position slot, or the Folium map HTML regeneration is not triggering the web view reload. Fix: (1) Confirm the connection in MapWidget.__init__: self.be.sig_position.connect(self._on_position) must be present. (2) Add a print statement in _on_position to confirm it's being called. (3) If the map tile fails to load (no internet), use tiles='CartoDB positron' or tiles=None in the folium.Map() call — the offline fallback shows a grey canvas but still shows markers and the flight track polyline.

Extension Challenges

  1. Replace the HTML regeneration map with a JavaScript-driven live update.
