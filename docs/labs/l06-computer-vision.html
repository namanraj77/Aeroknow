Vision Pipeline Concepts — What Runs Where and Why

The Threading Problem

YOLOv8n inference on a CPU takes 50–200 ms per frame. The camera publishes at 30 Hz — a new frame every 33 ms. If you run inference synchronously inside the ROS2 callback, the callback takes 150 ms, blocking every other callback in the node for that time. The ROS2 executor falls behind, the topic queue backs up, and the node processes frames from seconds ago — useless for real-time detection.

The Solution: Inference in a Thread Pool The image callback does one thing only: copy the latest frame into a shared buffer and return immediately (microseconds). A separate Python thread constantly pulls from that buffer and runs YOLOv8n inference. When inference finishes, it posts results to an output queue. The ROS2 publish timer reads the output queue and publishes detections. This decouples camera rate (30 Hz) from inference rate (~5–15 Hz on CPU) without either blocking the other. ROS2 callbacks stay fast; inference runs at its natural speed.

The Four Key Concepts

Detection2DArray

The ROS2 standard message for a list of 2D detections. Each Detection2D contains a bounding box (centre x/y + width/height in pixels), a class ID, a confidence score, and a header with timestamp. Using the standard message type means the L07 GCS and L08 swarm coordinator can subscribe without knowing YOLOv8 was used — they just see detections.

Pinhole Projection

A pixel at (u, v) in the image corresponds to a ray in 3D space through the camera centre. The camera intrinsic matrix K maps between pixel coordinates and normalised image coordinates. Combined with drone altitude, this gives a ground-plane intersection — the world position of the detected object. Accuracy depends on the assumption that the target is on the ground plane.

Confidence Threshold

YOLOv8 outputs a confidence score for each detection. Low threshold (0.25) = more detections, more false positives. High threshold (0.7) = fewer detections, fewer false positives. For ISR target geolocation you want high precision (few false geolocations), so the threshold is set higher (0.5+). For L08 swarm, lower threshold gives more detections for the allocation algorithm to work with.

CameraInfo

The sensor_msgs/CameraInfo topic (bridged from Gazebo in L04) contains the camera intrinsic matrix K, distortion coefficients D, and image dimensions. The tracker reads this once at startup — it is the calibration that makes the pixel→world projection accurate. Without it, you must hardcode K values from the SDF FOV parameters.

Step 03: vision_node.py

The core detection node. Subscribes to the camera topic, runs YOLOv8n on each frame in a background thread, and publishes results as Detection2DArray messages. The threading model is the most important design decision here.

STEP 03 Write vision_node.py
Why the inference runs in a concurrent.futures ThreadPoolExecutor

Python's GIL means CPU-bound work in a pure Python thread doesn't parallelise well. However, YOLOv8 (via PyTorch) releases the GIL during tensor operations, so a ThreadPoolExecutor with one worker does give true CPU parallelism for inference. The image callback acquires a lock, replaces the pending frame, and returns — it never blocks on inference. The worker thread picks up the latest frame, runs YOLO, and posts results. If inference takes 200 ms and frames arrive at 33 ms, some frames are skipped — this is intentional and correct. Processing stale frames is worse than skipping them.

python · vision_node.py 
#!/usr/bin/env python3
"""
vision_node.py
──────────────
Runs YOLOv8n on the live Gazebo camera feed.
Publishes detections as vision_msgs/Detection2DArray.
Uses a background thread for inference to keep ROS2 callbacks fast.

Published topics:
  /vision/detections  (vision_msgs/Detection2DArray) — YOLO results
  /vision/fps         (std_msgs/Float32)              — inference FPS
  /vision/annotated   (sensor_msgs/Image)             — annotated frame
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from sensor_msgs.msg  import Image
from std_msgs.msg     import Float32
from vision_msgs.msg  import Detection2DArray, Detection2D, ObjectHypothesisWithPose, BoundingBox2D

import numpy as np
import cv2
import threading
import time
from concurrent.futures import ThreadPoolExecutor
from queue import Queue, Empty

try:
    from cv_bridge import CvBridge
    CV_BRIDGE_OK = True
except ImportError:
    CV_BRIDGE_OK = False
    print("[WARN] cv_bridge not found — install ros-humble-cv-bridge")

from ultralytics import YOLO

CONF_THRESHOLD  = 0.45   # minimum confidence to publish a detection
IOU_THRESHOLD   = 0.45   # NMS IoU threshold (from Module 07 §7.4)
MODEL_PATH      = 'yolov8n.pt'   # downloads automatically on first run
MAX_QUEUE_DEPTH = 2       # frames waiting for inference — drop older frames


class VisionNode(Node):

    def __init__(self):
        super().__init__('vision_node')

        # ── Load YOLOv8n model ───────────────────────────────────────
        self.get_logger().info(f'Loading YOLOv8n from {MODEL_PATH}...')
        self.model = YOLO(MODEL_PATH)
        self.get_logger().info('YOLOv8n loaded')

        # ── Thread-safe frame buffer ─────────────────────────────────
        # Only one frame is ever pending — we want the most recent frame,
        # not a queue of stale frames.
        self._frame_lock    = threading.Lock()
        self._pending_frame = None          # latest camera frame (numpy BGR)
        self._result_queue  = Queue(maxsize=MAX_QUEUE_DEPTH)
        self._running       = True

        self.bridge = CvBridge() if CV_BRIDGE_OK else None

        # ── Performance tracking ─────────────────────────────────────
        self._fps_window  = []   # list of recent inference durations [s]
        self._det_count   = 0   # total detections since start

        # ── QoS ──────────────────────────────────────────────────────
        sen_qos = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                             history=HistoryPolicy.KEEP_LAST, depth=1)

        # ── Subscriptions ────────────────────────────────────────────
        self.create_subscription(Image, '/camera/image_raw',
                                 self.image_cb, sen_qos)

        # ── Publishers ───────────────────────────────────────────────
        self.det_pub  = self.create_publisher(Detection2DArray,
            '/vision/detections', 10)
        self.fps_pub  = self.create_publisher(Float32,
            '/vision/fps', 10)
        self.ann_pub  = self.create_publisher(Image,
            '/vision/annotated', 10)

        # ── Inference thread ─────────────────────────────────────────
        self._executor = ThreadPoolExecutor(max_workers=1)
        self._executor.submit(self._inference_loop)

        # ── Publish timer: posts results from inference queue to ROS2 ─
        self.create_timer(0.05, self._publish_results)   # 20 Hz poll

        self.get_logger().info(
            f'VisionNode ready. Conf={CONF_THRESHOLD}, IoU={IOU_THRESHOLD}')

    # ── ROS2 image callback ──────────────────────────────────────────
    # MUST return quickly. Only copies the frame; does no processing.

    def image_cb(self, msg: Image):
        if not (CV_BRIDGE_OK and self.bridge):
            return
        try:
            frame = self.bridge.imgmsg_to_cv2(msg, desired_encoding='bgr8')
        except Exception:
            return
        # Replace pending frame atomically — we only care about the LATEST frame
        with self._frame_lock:
            self._pending_frame = (frame.copy(), msg.header)

    # ── Inference thread ─────────────────────────────────────────────
    # Runs in a separate thread. Polls for new frames, runs YOLOv8n,
    # posts results to the result queue for the ROS2 timer to pick up.

    def _inference_loop(self):
        while self._running:
            # Grab the latest pending frame
            with self._frame_lock:
                item = self._pending_frame
                self._pending_frame = None   # consume it

            if item is None:
                time.sleep(0.01)
                continue

            frame, header = item
            t0 = time.time()

            # Run YOLOv8n inference (releases GIL inside PyTorch)
            results = self.model.predict(
                frame,
                conf=CONF_THRESHOLD,
                iou=IOU_THRESHOLD,
                verbose=False,
            )

            dt = time.time() - t0
            self._fps_window.append(dt)
            if len(self._fps_window) > 10: self._fps_window.pop(0)

            # Parse YOLO results → list of dicts
            detections = []
            if results and len(results) > 0:
                r = results[0]
                for box in r.boxes:
                    xyxy = box.xyxy[0].cpu().numpy()
                    x1, y1, x2, y2 = xyxy
                    cx = float((x1 + x2) / 2)
                    cy = float((y1 + y2) / 2)
                    w  = float(x2 - x1)
                    h  = float(y2 - y1)
                    conf  = float(box.conf[0].cpu())
                    cls   = int(box.cls[0].cpu())
                    label = self.model.names[cls]
                    detections.append({
                        'cx': cx, 'cy': cy, 'w': w, 'h': h,
                        'conf': conf, 'class_id': cls, 'label': label,
                    })

            self._det_count += len(detections)

            # Build annotated image (draw boxes with OpenCV)
            annotated = frame.copy()
            for d in detections:
                x1 = int(d['cx'] - d['w']/2)
                y1 = int(d['cy'] - d['h']/2)
                x2 = int(d['cx'] + d['w']/2)
                y2 = int(d['cy'] + d['h']/2)
                cv2.rectangle(annotated, (x1,y1), (x2,y2), (0,255,0), 2)
                lbl = f"{d['label']} {d['conf']:.2f}"
                cv2.putText(annotated, lbl, (x1, y1-8),
                            cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0,255,0), 1)

            # Post to result queue (drop oldest if full)
            if self._result_queue.full():
                try: self._result_queue.get_nowait()
                except Empty: pass
            self._result_queue.put_nowait((header, detections, annotated, dt))

    # ── ROS2 publish timer ───────────────────────────────────────────
    # Runs at 20 Hz. Drains the result queue and publishes to ROS2 topics.

    def _publish_results(self):
        try:
            header, detections, annotated, dt = self._result_queue.get_nowait()
        except Empty:
            return

        stamp = self.get_clock().now().to_msg()

        # ── Publish Detection2DArray ──────────────────────────────────
        arr = Detection2DArray()
        arr.header.stamp = stamp
        arr.header.frame_id = 'camera_link'

        for d in detections:
            det = Detection2D()
            det.header = arr.header

            # Bounding box (centre + size in pixels)
            det.bbox.center.position.x = d['cx']
            det.bbox.center.position.y = d['cy']
            det.bbox.size_x = d['w']
            det.bbox.size_y = d['h']

            # Hypothesis (class + confidence)
            hyp = ObjectHypothesisWithPose()
            hyp.hypothesis.class_id = str(d['class_id'])
            hyp.hypothesis.score    = d['conf']
            det.results.append(hyp)
            arr.detections.append(det)

        self.det_pub.publish(arr)

        # ── Publish annotated image ───────────────────────────────────
        if self.bridge:
            try:
                img_msg = self.bridge.cv2_to_imgmsg(annotated, encoding='bgr8')
                img_msg.header.stamp = stamp
                self.ann_pub.publish(img_msg)
            except Exception: pass

        # ── Publish inference FPS ────────────────────────────────────
        if self._fps_window:
            fps = 1.0 / (sum(self._fps_window) / len(self._fps_window))
            fps_msg = Float32(); fps_msg.data = float(fps)
            self.fps_pub.publish(fps_msg)

        # ── Log summary (throttled) ───────────────────────────────────
        if detections:
            labels = [d['label'] for d in detections]
            self.get_logger().info(
                f'Detected: {labels} | {dt*1000:.0f}ms | total={self._det_count}',
                throttle_duration_sec=1.0
            )

    def destroy_node(self):
        self._running = False
        self._executor.shutdown(wait=False)
        super().destroy_node()


def main(args=None):
    rclpy.init(args=args)
    node = VisionNode()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

#!/usr/bin/env python3
"""
detection_visualiser.py
───────────────────────
Subscribes to /camera/image_raw + /vision/detections.
Draws bounding boxes, labels, confidence scores, and a HUD overlay.
Publishes annotated image to /vision/overlay for the L07 GCS.

Also opens a local OpenCV window if DISPLAY is available.
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from sensor_msgs.msg import Image
from vision_msgs.msg import Detection2DArray

import numpy as np
import cv2
import os
import time
from ultralytics.utils.plotting import colors as yolo_colors

try:
    from cv_bridge import CvBridge
    CV_BRIDGE_OK = True
except ImportError:
    CV_BRIDGE_OK = False

# COCO class names for label lookup (indices 0–79)
COCO_NAMES = {
    0:'person',1:'bicycle',2:'car',3:'motorbike',4:'aeroplane',
    5:'bus',6:'train',7:'truck',8:'boat',9:'traffic light',
    56:'chair',57:'couch',58:'potted plant',59:'bed',
    60:'dining table',62:'tv',63:'laptop',67:'cell phone',
}

HAS_DISPLAY = os.environ.get('DISPLAY') is not None


class DetectionVisualiser(Node):

    def __init__(self):
        super().__init__('detection_visualiser')
        self.bridge = CvBridge() if CV_BRIDGE_OK else None
        self.latest_dets = []
        self.frame_count = 0
        self.t_start = time.time()

        sen_qos = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                             history=HistoryPolicy.KEEP_LAST, depth=1)
        rel_qos = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                             history=HistoryPolicy.KEEP_LAST, depth=10)

        self.create_subscription(Image, '/camera/image_raw',
                                 self.image_cb, sen_qos)
        self.create_subscription(Detection2DArray, '/vision/detections',
                                 self.det_cb, rel_qos)

        self.overlay_pub = self.create_publisher(Image, '/vision/overlay', 10)

        self.get_logger().info(
            f'DetectionVisualiser ready. Display={HAS_DISPLAY}')

    def det_cb(self, msg: Detection2DArray):
        """Cache the latest detections for overlay on the next frame."""
        self.latest_dets = msg.detections

    def image_cb(self, msg: Image):
        if not (CV_BRIDGE_OK and self.bridge):
            return
        try:
            frame = self.bridge.imgmsg_to_cv2(msg, desired_encoding='bgr8')
        except Exception:
            return

        overlay = frame.copy()
        h, w = overlay.shape[:2]

        # ── Draw detections ───────────────────────────────────────────
        for det in self.latest_dets:
            cx = det.bbox.center.position.x
            cy = det.bbox.center.position.y
            bw = det.bbox.size_x
            bh = det.bbox.size_y
            x1, y1 = int(cx - bw/2), int(cy - bh/2)
            x2, y2 = int(cx + bw/2), int(cy + bh/2)

            cls_id = int(det.results[0].hypothesis.class_id) if det.results else 0
            conf   = det.results[0].hypothesis.score          if det.results else 0
            label  = COCO_NAMES.get(cls_id, f'cls{cls_id}')

            # Colour per class (YOLO colour palette)
            colour = yolo_colors(cls_id, bgr=True)

            # Bounding box
            cv2.rectangle(overlay, (x1,y1), (x2,y2), colour, 2)

            # Label background + text
            label_str = f'{label} {conf:.2f}'
            (tw, th), _ = cv2.getTextSize(label_str, cv2.FONT_HERSHEY_SIMPLEX, 0.45, 1)
            cv2.rectangle(overlay, (x1, y1-th-6), (x1+tw+4, y1), colour, -1)
            cv2.putText(overlay, label_str, (x1+2, y1-4),
                        cv2.FONT_HERSHEY_SIMPLEX, 0.45, (0,0,0), 1)

            # Bounding box area as rough distance proxy
            area_pct = (bw * bh) / (w * h) * 100
            cv2.putText(overlay, f'{area_pct:.1f}%', (x1+2, y2+14),
                        cv2.FONT_HERSHEY_SIMPLEX, 0.4, colour, 1)

        # ── HUD overlay ───────────────────────────────────────────────
        self.frame_count += 1
        elapsed = time.time() - self.t_start
        display_fps = self.frame_count / elapsed if elapsed > 0 else 0
        n_det = len(self.latest_dets)

        cv2.putText(overlay, f'FPS: {display_fps:.1f}',
                    (10, 20), cv2.FONT_HERSHEY_SIMPLEX, 0.55, (0,255,0), 1)
        cv2.putText(overlay, f'Detections: {n_det}',
                    (10, 42), cv2.FONT_HERSHEY_SIMPLEX, 0.55, (0,255,0), 1)
        cv2.putText(overlay, 'AEROKNOW L06',
                    (w-130, 20), cv2.FONT_HERSHEY_SIMPLEX, 0.45, (0,200,255), 1)

        # ── Publish overlay ───────────────────────────────────────────
        try:
            out_msg = self.bridge.cv2_to_imgmsg(overlay, encoding='bgr8')
            out_msg.header.stamp = self.get_clock().now().to_msg()
            self.overlay_pub.publish(out_msg)
        except Exception: pass

        # ── Optional: show in local window ────────────────────────────
        if HAS_DISPLAY:
            cv2.imshow('AEROKNOW Vision', overlay)
            cv2.waitKey(1)


def main(args=None):
    rclpy.init(args=args)
    node = DetectionVisualiser()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally:
        if HAS_DISPLAY: cv2.destroyAllWindows()
        node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

Step 05: target_tracker.py — Pixel to World-Frame GPS

The most geometrically interesting node. Given a detection bounding box centre in pixel coordinates, the drone's GPS position and altitude, and the camera intrinsic matrix, project the pixel ray through the camera model to a ground-plane intersection and compute the estimated GPS coordinates of the detected object.

The Projection Geometry

The pinhole camera model (Module 05 §5.3) gives us the ray from the camera centre through a pixel. For a downward-facing camera above a flat ground plane, the intersection of this ray with the ground gives the target's position:

Pixel to Ground Position — Step by Step 1. Convert pixel (u, v) to normalised image coordinates: (x_n, y_n) = ((u - cx)/fx, (v - cy)/fy) using the camera intrinsic matrix K. 2. This gives a unit direction vector in the camera frame: d_cam = [x_n, y_n, 1] (pointing toward the pixel). 3. Rotate to the world frame using the drone's attitude quaternion from MAVROS2. 4. Scale the ray to reach the ground plane: the scale factor is altitude / d_world.z (for a downward camera, d_world.z ≈ 1). 5. Add the drone's GPS position (converted to local Cartesian) to get target GPS. The accuracy of this depends on: altitude accuracy (barometer error ±0.5 m → position error ±0.5 × tan(FOV/2)), attitude accuracy (1° roll error at 30 m altitude → 0.5 m position error), and the assumption that the target is on the ground plane (invalid for elevated targets).

#!/usr/bin/env python3
"""
target_tracker.py
─────────────────
Projects pixel detections to GPS world-frame coordinates.
Uses pinhole camera model + drone attitude + altitude.

Subscriptions:
  /vision/detections           (vision_msgs/Detection2DArray)
  /mavros/local_position/pose  (geometry_msgs/PoseStamped)
  /mavros/global_position/global (sensor_msgs/NavSatFix)
  /camera/camera_info          (sensor_msgs/CameraInfo)

Published:
  /targets/gps   (geographic_msgs/GeoPoint) — estimated GPS of latest detection
  /targets/local (geometry_msgs/PointStamped) — ENU local frame position
"""

import rclpy
from rclpy.node import Node
from rclpy.qos  import QoSProfile, ReliabilityPolicy, HistoryPolicy

from sensor_msgs.msg    import CameraInfo, NavSatFix
from geometry_msgs.msg  import PoseStamped, PointStamped
from vision_msgs.msg    import Detection2DArray
from geographic_msgs.msg import GeoPoint

import numpy as np
import math


class TargetTracker(Node):

    def __init__(self):
        super().__init__('target_tracker')

        # ── Camera intrinsics (populated from /camera/camera_info) ────
        # K = [[fx, 0, cx], [0, fy, cy], [0, 0, 1]]
        # Default values from the SDF: 80° FOV, 640×480
        # fx = fy = (image_width / 2) / tan(FOV/2)
        fov_rad = 1.3963   # 80° in radians (from model.sdf)
        self.fx = (640 / 2) / math.tan(fov_rad / 2)
        self.fy = self.fx
        self.cx = 320.0   # principal point x (image centre)
        self.cy = 240.0   # principal point y (image centre)
        self.camera_info_received = False

        # ── Drone state ──────────────────────────────────────────────
        self.drone_lat  = None
        self.drone_lon  = None
        self.drone_alt  = None   # ellipsoidal altitude [m]
        self.altitude_m = 1.0   # local AGL altitude from pose [m]
        self.q_w = 1.0; self.q_x = 0.0; self.q_y = 0.0; self.q_z = 0.0

        sen_qos = QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT,
                             history=HistoryPolicy.KEEP_LAST, depth=1)
        rel_qos = QoSProfile(reliability=ReliabilityPolicy.RELIABLE,
                             history=HistoryPolicy.KEEP_LAST, depth=10)

        # ── Subscriptions ────────────────────────────────────────────
        self.create_subscription(Detection2DArray, '/vision/detections',
            self.det_cb, rel_qos)
        self.create_subscription(PoseStamped, '/mavros/local_position/pose',
            self.pose_cb, sen_qos)
        self.create_subscription(NavSatFix, '/mavros/global_position/global',
            self.gps_cb, sen_qos)
        self.create_subscription(CameraInfo, '/camera/camera_info',
            self.caminfo_cb, rel_qos)

        # ── Publishers ───────────────────────────────────────────────
        self.gps_pub   = self.create_publisher(GeoPoint, '/targets/gps', 10)
        self.local_pub = self.create_publisher(PointStamped, '/targets/local', 10)

        self.get_logger().info('TargetTracker ready')

    def pose_cb(self, msg: PoseStamped):
        self.altitude_m = max(0.5, msg.pose.position.z)
        # Store attitude quaternion for projection
        self.q_w = msg.pose.orientation.w
        self.q_x = msg.pose.orientation.x
        self.q_y = msg.pose.orientation.y
        self.q_z = msg.pose.orientation.z

    def gps_cb(self, msg: NavSatFix):
        self.drone_lat = msg.latitude
        self.drone_lon = msg.longitude
        self.drone_alt = msg.altitude

    def caminfo_cb(self, msg: CameraInfo):
        """Update intrinsics from actual calibration data if available."""
        if msg.k[0] > 0 and not self.camera_info_received:
            self.fx = msg.k[0]
            self.fy = msg.k[4]
            self.cx = msg.k[2]
            self.cy = msg.k[5]
            self.camera_info_received = True
            self.get_logger().info(
                f'Camera intrinsics updated: fx={self.fx:.1f} fy={self.fy:.1f} '
                f'cx={self.cx:.1f} cy={self.cy:.1f}')

    def _quat_to_rot_matrix(self, qw, qx, qy, qz) -> np.ndarray:
        """Convert unit quaternion to 3×3 rotation matrix (ENU body frame)."""
        r = np.array([
            [1 - 2*(qy*qy+qz*qz),   2*(qx*qy-qw*qz),       2*(qx*qz+qw*qy)],
            [2*(qx*qy+qw*qz),       1 - 2*(qx*qx+qz*qz),   2*(qy*qz-qw*qx)],
            [2*(qx*qz-qw*qy),       2*(qy*qz+qw*qx),       1 - 2*(qx*qx+qy*qy)],
        ])
        return r

    def pixel_to_world_enu(self, u: float, v: float) -> tuple:
        """
        Project pixel (u, v) to ENU world position given current drone state.
        Returns (east_m, north_m, 0) relative to drone's home position,
        or None if insufficient data.
        """
        if self.drone_lat is None:
            return None

        # Step 1: pixel → normalised camera coordinates
        # (u - cx) / fx = x_n, (v - cy) / fy = y_n
        # Camera is mounted facing DOWN (pitch = +90° in model.sdf),
        # so the camera Z-axis points downward (-Z_body)
        x_n = (u - self.cx) / self.fx
        y_n = (v - self.cy) / self.fy

        # Step 2: direction vector in camera frame
        # Camera frame: X right, Y down, Z forward (into ground for downward cam)
        d_cam = np.array([x_n, y_n, 1.0])
        d_cam /= np.linalg.norm(d_cam)

        # Step 3: transform to body frame
        # For a camera mounted with pitch +90° (facing down):
        # camera X → body X (East)
        # camera Y → body -Z (Down in body = Up in world reversed)
        # camera Z → body -Y (South in ENU)
        # Rotation from camera to body frame (pitch 90° about Y-axis):
        R_cam_to_body = np.array([
            [1,  0,  0],
            [0,  0, -1],
            [0,  1,  0],
        ])
        d_body = R_cam_to_body @ d_cam

        # Step 4: transform body → ENU world using drone attitude
        R_body_to_world = self._quat_to_rot_matrix(
            self.q_w, self.q_x, self.q_y, self.q_z)
        d_world = R_body_to_world @ d_body

        # Step 5: intersect with ground plane (Z=0 in ENU)
        # Ray: P = drone_pos + t * d_world
        # Ground: Z = 0 → t = -altitude / d_world.z
        if abs(d_world[2]) < 1e-4:
            return None   # ray is nearly horizontal — no ground intersection

        t = -self.altitude_m / d_world[2]
        if t < 0:
            return None   # intersection is behind the camera

        # Target position in ENU relative to drone (at drone's home origin)
        target_east  = t * d_world[0]   # metres East
        target_north = t * d_world[1]   # metres North

        return target_east, target_north

    def _enu_offset_to_gps(self, east_m: float, north_m: float) -> tuple:
        """
        Convert ENU offset (metres) from drone GPS position to absolute GPS.
        Uses flat-Earth approximation — valid for offsets < 1 km.
        d_lat = north_m / (R * pi/180)
        d_lon = east_m  / (R * cos(lat) * pi/180)
        """
        R = 6_371_000   # Earth radius [m]
        lat_deg = self.drone_lat
        lon_deg = self.drone_lon

        d_lat = north_m / (R * math.pi / 180)
        d_lon = east_m  / (R * math.cos(math.radians(lat_deg)) * math.pi / 180)

        return lat_deg + d_lat, lon_deg + d_lon

    def det_cb(self, msg: Detection2DArray):
        """
        For each detection, project bounding box centre to GPS.
        Publish the highest-confidence detection's GPS position.
        """
        if not msg.detections:
            return

        # Pick the highest confidence detection
        best = max(msg.detections,
                   key=lambda d: d.results[0].hypothesis.score if d.results else 0)

        u = best.bbox.center.position.x
        v = best.bbox.center.position.y

        result = self.pixel_to_world_enu(u, v)
        if result is None:
            return

        east_m, north_m = result

        # GPS position
        target_lat, target_lon = self._enu_offset_to_gps(east_m, north_m)

        gps_msg = GeoPoint()
        gps_msg.latitude  = target_lat
        gps_msg.longitude = target_lon
        gps_msg.altitude  = 0.0   # ground level
        self.gps_pub.publish(gps_msg)

        # Local ENU position (for GCS map display)
        local_msg = PointStamped()
        local_msg.header.stamp    = self.get_clock().now().to_msg()
        local_msg.header.frame_id = 'map'
        local_msg.point.x = east_m
        local_msg.point.y = north_m
        local_msg.point.z = 0.0
        self.local_pub.publish(local_msg)

        conf = best.results[0].hypothesis.score if best.results else 0
        cls  = best.results[0].hypothesis.class_id if best.results else '?'
        self.get_logger().info(
            f'Target cls={cls} conf={conf:.2f} | '
            f'GPS ({target_lat:.6f}, {target_lon:.6f}) | '
            f'ENU offset: E={east_m:.1f}m N={north_m:.1f}m | '
            f'alt={self.altitude_m:.1f}m',
            throttle_duration_sec=1.0
        )


def main(args=None):
    rclpy.init(args=args)
    node = TargetTracker()
    try: rclpy.spin(node)
    except KeyboardInterrupt: pass
    finally: node.destroy_node(); rclpy.shutdown()

if __name__ == '__main__': main()

Common Errors & Fixes

Error 01 — YOLOv8n detects nothing in the Gazebo scene vision_node: (no detection log lines appear, /vision/detections publishes empty arrays)

Cause: The red barrel from L04 is not a COCO class — YOLOv8n does not have a "barrel" class to detect. The model will only fire on objects that resemble its 80 trained classes. The barrel will not be detected as-is. Fix: Use the world_spawner to spawn a vehicle-shaped box as in Step 02 — Gazebo box primitives with vehicle-like aspect ratios (~4m × 2m × 1.5m) are sometimes classified as "car" or "truck" by YOLO. If detections are still sparse, lower CONF_THRESHOLD to 0.25. For reliable detection of custom targets, see the fine-tuning extension challenge below.

Error 02 — Inference rate is very low (<2 Hz) /vision/fps publishes values of 1.2 Hz or lower

Cause: CPU is the bottleneck — YOLOv8n on CPU is 50–200 ms per frame depending on the machine. On a dual-core VM, 1–2 Hz is expected. Fix: (1) Use a smaller input resolution — add imgsz=320 to model.predict() call. This halves inference time with modest accuracy loss. (2) Set device='cpu' explicitly and enable PyTorch inference mode: wrap the predict call in with torch.inference_mode():. (3) If you have a GPU, pass device=0 — inference will jump to 30+ Hz. Detection and geolocation quality is not affected by inference rate — each frame is still processed correctly, just less frequently.

Error 03 — target_tracker reports very large GPS offset ENU offset: E=150.3m N=89.2m (far from actual target position)

Cause: The drone attitude quaternion is not being received correctly (all zeros), so the projection uses an identity rotation — correct only when the drone is perfectly level. Any roll or pitch (from wind or manoeuvre) tilts the camera and shifts the projected ground point. Fix: (1) Confirm /mavros/local_position/pose is publishing: ros2 topic hz /mavros/local_position/pose should show ~30 Hz. (2) Check the quaternion fields in the pose message — ros2 topic echo /mavros/local_position/pose --once should show orientation.w ≈ 1.0 when hovering level. (3) The projection assumes the target is on the ground plane (z=0) — if the drone is banked, the calculated intersection will be wrong until the drone levels out. Test with the drone in a stable hover before checking geolocation accuracy.

Error 04 — /vision/overlay topic exists but image is black rqt image_view shows all-black frame on /vision/overlay

Cause: The cv_bridge encoding mismatch — the Gazebo camera publishes in rgb8 but the bridge may deliver bgr8 or vice versa, causing colour channel swap and appearing black in some viewers. Fix: In detection_visualiser.py, change the imgmsg_to_cv2 encoding from 'bgr8' to 'rgb8' (or use 'passthrough' and check the actual encoding with ros2 topic echo /camera/image_raw --once | grep encoding). Match the cv2_to_imgmsg encoding output to the same value.

Error 05 — ModuleNotFoundError: ultralytics.utils.plotting ImportError: cannot import name 'colors' from 'ultralytics.utils.plotting'

Cause: The colors function moved between Ultralytics versions. The import path differs between ultralytics <8.0 and ≥8.0. Fix: Update to the latest version: pip3 install --upgrade ultralytics. Alternatively, replace the import with a hardcoded colour list: replace the yolo_colors(cls_id, bgr=True) call with ((0,255,0) if cls_id % 2 == 0 else (0,165,255)) — a simple two-colour fallback.

Extension Challenges

The vision pipeline is running. Here are three harder things to try before L07.

  1. Fine-tune YOLOv8n to detect the red barrel. The red barrel (a red cylinder in Gazebo) is not a COCO class — zero-shot detection fails on it. Using Ultralytics' training API, create a minimal dataset: fly the drone over the barrel, record 50–100 frames from /camera/image_raw, label them with a single "barrel" class using Roboflow or LabelImg, and fine-tune with model.train(data='barrel.yaml', epochs=20, imgsz=320). A fine-tuned model on even 50 examples should achieve P>0.8 on the Gazebo barrel. This is the sim-to-real transfer problem from Module 07 §7.6 — minimised when the simulation renderer matches the real sensor as closely as possible.
  2. Add multi-target tracking with an EKF per target. The current tracker geolocates one target per frame and publishes immediately. A real ISR system tracks each target over time to smooth noisy detections. Implement a simple multi-target tracker: maintain a dict of target_id → (lat, lon, last_seen_t, kalman_state). Each new detection is matched to the nearest existing track (within 5 m) or starts a new track. Use a 2-state Kalman filter (position only) per track. Publish the smoothed GPS position — noise should drop from ±3 m instantaneous to ±0.5 m after 10+ observations.
  3. Implement an orbit-and-detect mission behaviour. Extend flight_controller.py (from L03 Extension Challenge 1) with a new state: ORBIT. When the vision node detects a target, the flight controller should transition to ORBIT and use velocity setpoints to fly a 15 m radius circle centred on the detected target's GPS position. During the orbit, the target tracker continuously refines the geolocation estimate. After one full orbit (360°), transition to LANDING. This is the standard ISR engagement geometry — stand-off circle at constant altitude maximises dwell time and geolocation accuracy.
